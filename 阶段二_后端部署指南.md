## 2.2 安装Python依赖

```bash
# 第1步：进入后端目录
cd /opt/juyunbang/code/backend

# 第2步：创建Python虚拟环境
python3 -m venv venv

# 第3步：激活虚拟环境
source venv/bin/activate

# 第4步：升级pip（避免安装报错）
pip install --upgrade pip

# 第5步：安装所有依赖（这一步需要几分钟，耐心等待）
pip install -r requirements.txt
```

**等待安装完成后，看到 `Successfully installed ...` 就是成功了。**

如果安装过程中出现红色报错提到 `lxml` 或 `cryptography`，执行以下命令后重试：
```bash
apt install -y libxml2-dev libxslt1-dev libffi-dev build-essential
pip install -r requirements.txt
```

## 2.3 创建数据库表

```bash
# 第1步：确保你在后端目录
cd /opt/juyunbang/code/backend

# 第2步：创建所有数据库表
mysql -u juyunbang -p juyunbang < migrations/schema.sql
# 输入你在阶段一步骤1.3中为 juyunbang 用户设置的数据库密码
# 注意：这里的密码是数据库用户juyunbang的密码，不是root密码

# 第3步：导入测试数据（这样小程序打开就能看到数据）
mysql -u juyunbang -p juyunbang < migrations/seed_data.sql
# 再次输入同样的密码
```

**验证数据库是否创建成功：**
```bash
mysql -u juyunbang -p -e "USE juyunbang; SHOW TABLES;"
# 输入密码后，应该看到18张表的列表
```

## 2.4 配置环境变量

代码里已经有一个 `.env` 文件，但你需要确认里面的密码等信息是否正确。

```bash
# 查看当前配置
cat /opt/juyunbang/code/backend/.env
```

**你需要核对以下几项，确保跟你的实际情况一致：**

| 配置项 | 说明 | 需要核对 |
|--------|------|---------|
| DB_PASSWORD | 数据库密码 | 必须与步骤1.3中创建juyunbang用户时设置的密码一致 |
| WX_APPID | 微信小程序AppID | 在微信公众平台后台获取 |
| WX_SECRET | 微信小程序AppSecret | 在微信公众平台后台获取 |
| QINIU_ACCESS_KEY | 七牛云AK | 在七牛云个人中心-密钥管理获取 |
| QINIU_SECRET_KEY | 七牛云SK | 在七牛云个人中心-密钥管理获取 |
| QINIU_DOMAIN | 七牛云域名 | 你绑定的CDN域名 |

**如果需要修改，执行：**
```bash
vim /opt/juyunbang/code/backend/.env
```

> **vim编辑器使用方法（最简单版）：**
> 1. 打开文件后，按 `i` 键进入编辑模式（左下角显示 `-- INSERT --`）
> 2. 用方向键移到要改的位置，删除旧内容，输入新内容
> 3. 改完后按 `Esc` 键退出编辑模式
> 4. 输入 `:wq` 然后按回车 → 保存并退出
> 5. 如果改错了想放弃，按 `Esc` 后输入 `:q!` 然后按回车 → 不保存退出

## 2.7 配置Gunicorn + Systemd（让后端在后台持续运行）

```bash
# 创建systemd服务文件
cat > /etc/systemd/system/juyunbang.service << 'EOF'
[Unit]
Description=JuYunBang Backend API
After=network.target mariadb.service redis-server.service

[Service]
Type=notify
User=root
Group=root
WorkingDirectory=/opt/juyunbang/code/backend
Environment="PATH=/opt/juyunbang/code/backend/venv/bin:/usr/bin"
ExecStart=/opt/juyunbang/code/backend/venv/bin/gunicorn -c gunicorn_config.py run:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

```bash
# 重新加载systemd配置
systemctl daemon-reload

# 启动后端服务
systemctl start juyunbang

# 设置开机自动启动
systemctl enable juyunbang

# 查看服务状态（应该显示 active (running)）
systemctl status juyunbang
```

**你应该看到绿色的 `active (running)` 字样。** 如果显示红色的 `failed`，执行以下命令查看详细错误：
```bash
journalctl -u juyunbang -n 50 --no-pager
```
把输出复制给我，我帮你排查。

## 常见问题

### Q: `mysql -u juyunbang -p` 报错 "Access denied"
A: 密码输错了。如果忘记了密码，用root重新设置：
```bash
mysql -u root -p
# 输入root密码后执行：
ALTER USER 'juyunbang'@'localhost' IDENTIFIED BY '你的新密码';
FLUSH PRIVILEGES;
EXIT;
```
然后更新 `/opt/juyunbang/code/backend/.env` 中的 `DB_PASSWORD`。

### Q: `pip install` 特别慢
A: 使用国内镜像加速：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q: `systemctl status juyunbang` 显示 failed
A: 执行 `journalctl -u juyunbang -n 50 --no-pager` 查看详细错误，把输出复制给我。

### Q: `curl https://api.sqnl8.cn/...` 报错
A: 检查Nginx配置：
```bash
nginx -t
systemctl restart nginx
```
如果还是不行，把 `nginx -t` 的输出复制给我。
