## 5.2 采集服务部署

数据采集服务是一个独立运行的定时任务程序，负责定期从各平台抓取热度数据。它与后端API服务分开运行。

**采集任务时间表：**

| 任务 | 频率 | 说明 |
|------|------|------|
| 热度数据采集 | 每15分钟 | 爱奇艺/优酷/腾讯/芒果TV/B站 |
| 社交数据采集 | 每60分钟 | 微博/抖音/百度 |
| 数据清洗 | 每天00:00 | 去除异常值和重复记录 |
| 日度统计计算 | 每天00:30 | 汇总当日数据 |
| 剧力指数计算 | 每天01:00 | 综合指数排名 |
| 热度异动检测 | 每天01:30 | 检测热度突增的剧集 |
| 每日数据发布 | 每天15:00 | 发布日榜 + 生成每日简报 |
| 豆瓣评分更新 | 每天03:00 | 更新豆瓣评分 |
| 旧数据清理 | 每天04:00 | 清理30天前的实时数据 |

### 第1步：创建采集服务的systemd配置

在服务器上执行：

```bash
cat > /etc/systemd/system/juyunbang-crawler.service << 'EOF'
[Unit]
Description=JuYunBang Crawler Service
After=network.target mariadb.service redis-server.service

[Service]
Type=simple
User=root
Group=root
WorkingDirectory=/opt/juyunbang/code/backend
Environment="PATH=/opt/juyunbang/code/backend/venv/bin:/usr/bin"
Environment=PYTHONPATH=/opt/juyunbang/code/backend
ExecStart=/opt/juyunbang/code/backend/venv/bin/python scheduler/task_scheduler.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF
```

### 第2步：启动采集服务

```bash
# 重新加载systemd配置
systemctl daemon-reload

# 启动采集服务
systemctl start juyunbang-crawler

# 设置开机自动启动
systemctl enable juyunbang-crawler

# 查看服务状态（应该显示 active (running)）
systemctl status juyunbang-crawler
```

**你应该看到绿色的 `active (running)` 字样。**

如果显示 `failed`，查看详细错误：
```bash
journalctl -u juyunbang-crawler -n 50 --no-pager
```
把输出截图给我。

### 第3步：验证采集服务正在工作

等待约15分钟后（等第一次热度采集完成），查看是否有新数据：

```bash
# 激活虚拟环境
cd /opt/juyunbang/code/backend
source venv/bin/activate

# 检查采集任务日志
mysql -u juyunbang -p -e "SELECT * FROM juyunbang.crawl_tasks ORDER BY id DESC LIMIT 10;"

# 检查是否有实时热度数据
mysql -u juyunbang -p -e "SELECT COUNT(*) as total FROM juyunbang.heat_realtime WHERE DATE(created_at) = CURDATE();"
```

**常用采集服务管理命令：**
```bash
# 查看实时日志
journalctl -u juyunbang-crawler -f

# 重启采集服务
systemctl restart juyunbang-crawler

# 停止采集服务
systemctl stop juyunbang-crawler
```

## 5.3 微信后台配置服务器域名

小程序要能访问你的后端API，必须在微信公众平台后台配置合法域名。

### 第1步：登录微信公众平台

1. 打开 https://mp.weixin.qq.com
2. 用你注册小程序时的账号登录

### 第2步：配置服务器域名

1. 进入 **开发管理** → **开发设置** → **服务器域名**
2. 点击 **修改**（可能需要管理员扫码验证）
3. 在以下位置分别填入：

| 域名类型 | 填写内容 |
|---------|---------|
| request合法域名 | `https://api.sqnl8.cn` |
| uploadFile合法域名 | `https://api.sqnl8.cn` |
| downloadFile合法域名 | `https://api.sqnl8.cn` |

> **注意：** 域名必须是 `https://` 开头，且已完成ICP备案。你在阶段一已经配置了SSL证书，所以这里直接填即可。

4. 点击 **保存并提交**

### 第3步：记录AppID和AppSecret

在 **开发管理** → **开发设置** 页面：
1. 复制 **AppID**（小程序ID）
2. 点击 **AppSecret** 旁的"重置"或"查看"，获取AppSecret
3. 将这两个值更新到服务器的 `.env` 文件中：

```bash
vim /opt/juyunbang/code/backend/.env
# 修改 WX_APPID 和 WX_SECRET 为你的真实值
```

> **vim操作提示：** 按 `i` 进入编辑模式 → 修改内容 → 按 `Esc` → 输入 `:wq` 回车保存退出。如果改错了按 `Esc` 后输入 `:q!` 回车放弃修改。

修改后重启后端服务：
```bash
systemctl restart juyunbang
```

## 5.4 小程序前端设置与联调测试

### 第1步：下载微信开发者工具

1. 下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 安装后打开，用微信扫码登录

### 第2步：导入项目

1. 点击 **+** 号 → **导入项目**
2. **目录**：选择你电脑上项目的 `miniapp` 文件夹
   - 如果你之前是在服务器上 clone 的代码，需要在你的电脑上也 clone 一份：
   ```bash
   git clone https://github.com/nailao11/juyunbang.git
   ```
   然后选择 `juyunbang/miniapp` 目录
3. **AppID**：填入你在微信公众平台获取的真实 AppID
4. 点击 **确定**

### 第3步：修改项目配置

导入后，需要确认以下配置正确：

**检查 `project.config.json`：**
- 打开 `miniapp/project.config.json`
- 确认 `appid` 已改为你的真实 AppID（不是 `wxTODO_REPLACE_WITH_REAL_APPID`）

**检查 `app.js` 中的API地址：**
- 打开 `miniapp/app.js`
- 确认 `baseUrl` 为 `https://api.sqnl8.cn/api/v1`（应该已经是正确的）

### 第4步：联调测试

在微信开发者工具中：

1. **编译运行** — 点击上方"编译"按钮
2. **检查首页** — 应该能看到剧集热度排行列表（显示种子数据）
3. **测试各页面功能：**

| 页面 | 测试内容 | 预期结果 |
|------|---------|---------|
| 首页（热度） | 查看排行列表、切换平台 | 显示剧集列表和热度值 |
| 数据中心 | 查看日榜/播放榜 | 显示排行数据 |
| 发现 | 浏览分类、搜索 | 显示分类内容 |
| 我的 | 登录、查看个人信息 | 显示登录状态 |
| 剧集详情 | 点击任意剧集 | 显示详情和数据图表 |
| 搜索 | 输入关键词搜索 | 返回搜索结果 |

4. **查看调试面板** — 点击底部"调试器"，检查 Network 标签页，确认API请求返回 200

**如果出现网络错误：**
- 检查是否在 **详情** → **本地设置** 中勾选了"不校验合法域名"（开发阶段可以先勾选）
- 检查服务器后端服务是否正常运行：`systemctl status juyunbang`
- 检查 Nginx 是否正常：`nginx -t`

## 5.5 体验版测试

联调测试通过后，发布体验版给真机测试：

### 第1步：上传代码

1. 在微信开发者工具中，点击右上角 **上传**
2. 填写版本号：`1.0.0`
3. 填写项目备注：`首次提交，包含热度排行、数据中心、追剧管理等功能`
4. 点击 **上传**

### 第2步：设置体验版

1. 登录微信公众平台 https://mp.weixin.qq.com
2. 进入 **管理** → **版本管理**
3. 在"开发版本"中找到你刚上传的版本
4. 点击 **设为体验版**

### 第3步：添加体验成员

1. 在 **管理** → **成员管理** → **体验成员**
2. 添加你想让其测试的微信号（最多可添加数十人）

### 第4步：真机测试

1. 用微信扫描体验版二维码
2. 在真实手机上测试所有功能
3. 重点检查：
   - 页面加载速度
   - 数据显示是否正确
   - 微信登录是否正常
   - 图表是否正常渲染
   - 深色模式切换

## 5.6 提交微信审核

体验版测试没问题后，提交正式审核：

### 第1步：提交审核

1. 在微信公众平台 → **管理** → **版本管理**
2. 在"开发版本"中，点击 **提交审核**
3. 填写审核信息：
   - **功能页面**：选择首页（`pages/index/index`）
   - **标题**：剧云榜
   - **类目**：工具 → 信息查询 （或 生活服务 → 生活服务）
   - **标签**：影视、数据、追剧
4. 点击 **提交审核**

### 第2步：等待审核

- 审核通常需要 1-7 个工作日
- 可以在微信公众平台查看审核状态
- 如果审核被拒，会告诉你原因，按要求修改后重新提交

---

## 5.7 正式发布

审核通过后：

1. 在微信公众平台 → **管理** → **版本管理**
2. 在"审核版本"中，点击 **发布**
3. 确认发布

发布后，用户可以在微信中搜索"剧云榜"找到并使用你的小程序。

## 常用运维命令速查

```bash
# ===== 服务管理 =====
systemctl status juyunbang          # 查看API服务状态
systemctl status juyunbang-crawler  # 查看采集服务状态
systemctl restart juyunbang         # 重启API服务
systemctl restart juyunbang-crawler # 重启采集服务

# ===== 日志查看 =====
journalctl -u juyunbang -f          # 实时查看API日志
journalctl -u juyunbang-crawler -f  # 实时查看采集日志
tail -f /opt/juyunbang/logs/access.log  # Gunicorn访问日志
tail -f /opt/juyunbang/logs/error.log   # Gunicorn错误日志

# ===== 数据库 =====
mysql -u juyunbang -p juyunbang     # 进入数据库

# ===== Nginx =====
nginx -t                            # 测试Nginx配置
systemctl restart nginx             # 重启Nginx

# ===== 代码更新 =====
cd /opt/juyunbang/code
git pull                            # 拉取最新代码
systemctl restart juyunbang         # 重启API
systemctl restart juyunbang-crawler # 重启采集
```
