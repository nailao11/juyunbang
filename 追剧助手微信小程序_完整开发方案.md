## 第一部分：项目总览与定位

### 1.1 项目名称

「剧力场」/ 「追剧雷达」/ 「剧热榜」（建议三选一）

### 1.2 项目定位

一款面向追剧爱好者和影视行业从业者的微信小程序工具，提供全平台剧集/综艺的实时热度监控、播放量统计、历史趋势分析、个人追剧管理等一站式服务。

### 1.3 目标用户

- 普通追剧用户：想知道什么剧最火、值不值得看
- 影视行业从业者：编剧、制片人、宣传人员需要数据参考
- 影视自媒体：需要数据支撑内容创作
- 粉丝群体：关注偶像作品的热度表现
- 数据爱好者：喜欢分析对比各剧表现

### 1.4 对标产品分析

| 对标产品 | 我们的差异化 |
|---------|------------|
| 云合数据 | 云合偏B端收费，我们免费面向C端 |
| 酷云数据 | 酷云侧重收视率，我们全平台覆盖 |
| 猫眼专业版 | 猫眼偏电影，我们深耕剧集+综艺 |
| 骨朵数据 | 骨朵更新慢，我们实时+次日双模式 |
| Vlinkage | V榜已停更，市场空缺 |
| 灯塔专业版 | 灯塔是阿里系，数据有倾向性 |

### 3.2 数据采集技术方案

**方案A：网页数据解析（主要方案）**

技术栈：Python + Requests + BeautifulSoup/Scrapy + Selenium

具体实现：

```
1. 爱奇艺热度：
   - 来源页面：爱奇艺风云榜 https://www.iqiyi.com/ranks
   - 数据位置：页面HTML中的热度值数字
   - 采集频率：每15分钟一次
   - 注意：爱奇艺热度值是其站内算法生成的，直接读取即可

2. 优酷热度：
   - 来源页面：优酷热播榜
   - 采集频率：每15分钟一次

3. 腾讯视频热度：
   - 来源页面：腾讯视频热搜榜
   - 数据接口：腾讯视频有公开的H5接口
   - 采集频率：每15分钟一次

4. 芒果TV热度：
   - 来源页面：芒果TV热播榜
   - 采集频率：每15分钟一次

5. 微博数据：
   - 来源：微博超话主页、话题页面
   - 数据：阅读量、讨论量
   - 采集频率：每30分钟一次

6. 抖音数据：
   - 来源：抖音话题页面
   - 数据：话题播放量
   - 采集频率：每小时一次

7. 百度指数：
   - 来源：百度指数官网
   - 需要登录cookie
   - 采集频率：每日一次（百度指数按天更新）
```

### 3.3 播放量统计计算方法

```
┌──────────────────────────────────────────────────────────────┐
│                     日播放量计算方法                            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  ● 基础公式：                                                 │
│    日播放增量 = 当日最后一次采集的累计播放量                      │
│              - 前一日最后一次采集的累计播放量                     │
│                                                              │
│  ● 采集时间节点：                                              │
│    每日采集截止时间：次日 00:00（即自然日0-24点）                 │
│    发布时间：次日 15:00                                        │
│    （中间时间用于数据清洗、异常检测、人工审核）                    │
│                                                              │
│  ● 数据清洗规则：                                              │
│    1. 异常值检测：日增量超过前7日均值的500%标记为异常              │
│    2. 回退检测：如果累计播放量比前次小，标记数据异常               │
│    3. 零值检测：如果某次采集值为0，使用前后值插值                  │
│    4. 去水处理：参考行业经验设定合理阈值                          │
│                                                              │
│  ● 多平台汇总：                                                │
│    全网播放量 = Σ 各平台日播放增量                               │
│    （对于独播剧，全网播放量 = 该平台播放量）                      │
│                                                              │
│  ● 有效播放量：                                                │
│    参考云合数据理念，可考虑引入"有效播放"概念                     │
│    有效播放 = 总播放量 × 有效系数                                │
│    有效系数根据多维度数据交叉验证得出                             │
│    （此为高级功能，初期可不实现）                                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 3.4 「剧力指数」综合评分算法

```
┌──────────────────────────────────────────────────────────────┐
│                    「剧力指数」计算公式                         │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  剧力指数（0-100分）= 各维度加权总分                            │
│                                                              │
│  计算维度及权重：                                              │
│  ┌───────────────────────────────────────────────────┐       │
│  │  维度1：平台热度（权重35%）                          │       │
│  │  计算方式：                                         │       │
│  │  - 取各播出平台的站内热度值                           │       │
│  │  - 标准化到0-100区间                                │       │
│  │  - 公式：(当前热度-最低值)/(最高值-最低值) × 100      │       │
│  │  - 多平台取加权平均                                  │       │
│  │                                                    │       │
│  │  维度2：全网讨论度（权重25%）                         │       │
│  │  计算方式：                                         │       │
│  │  - 微博话题增量：权重40%                             │       │
│  │  - 抖音话题增量：权重30%                             │       │
│  │  - 百度搜索指数：权重20%                             │       │
│  │  - 微信搜索指数：权重10%                             │       │
│  │  - 标准化到0-100区间                                │       │
│  │                                                    │       │
│  │  维度3：播放表现（权重25%）                           │       │
│  │  计算方式：                                         │       │
│  │  - 日播放量在同期剧集中的排名百分位                    │       │
│  │  - TOP1 = 100分，按百分位线性映射                     │       │
│  │                                                    │       │
│  │  维度4：口碑评价（权重15%）                           │       │
│  │  计算方式：                                         │       │
│  │  - 豆瓣评分 × 10（即8.5分 = 85分）                   │       │
│  │  - 未开分的取同类型剧集均值                           │       │
│  └───────────────────────────────────────────────────┘       │
│                                                              │
│  最终公式：                                                   │
│  剧力指数 = 平台热度得分×0.35 + 讨论度得分×0.25               │
│           + 播放表现得分×0.25 + 口碑得分×0.15                  │
│                                                              │
│  更新频率：                                                   │
│  - 实时版：每30分钟更新一次（不含播放量维度）                    │
│  - 日度版：次日15:00更新（包含所有维度）                        │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 4.3 推荐方案说明（为什么选Python）

```
对于零基础用户，推荐使用Python作为后端语言，原因：
1. Python语法简洁，零基础学习成本最低
2. 数据采集（爬虫）是Python最擅长的领域
3. 后端API和数据采集可以用同一种语言，减少学习负担
4. 数据处理/计算用Python的pandas库非常方便
5. Flask框架非常简单，几行代码就能启动一个API服务
6. 社区资源丰富，遇到问题容易找到解答

技术栈最终确定：
前端：微信小程序原生
后端：Python + Flask + MySQL + Redis
采集：Python + Requests/Selenium + APScheduler
```

## 第十部分：微信小程序注册与配置

### 10.1 注册步骤（详细图文级）

**步骤1：注册微信小程序账号**

```
1. 打开浏览器，访问 https://mp.weixin.qq.com
2. 点击右上角「立即注册」
3. 选择「小程序」类型
4. 填写注册信息：
   - 邮箱：填一个未注册过微信公众平台的邮箱
   - 密码：设置密码
   - 确认密码
5. 去邮箱查收验证邮件，点击链接激活
6. 选择主体类型：「个人」（个人主体有限制，但足够需求）
7. 填写个人信息：
   - 真实姓名
   - 身份证号
   - 手机号
   - 微信扫码验证管理员身份
8. 注册完成，进入小程序管理后台
```

**步骤2：完善小程序信息**

```
1. 登录小程序管理后台
2. 左侧菜单「设置」→「基本设置」
3. 填写：
   - 小程序名称：（如"剧力场"）
   - 小程序简称：（如"剧力场"）
   - 小程序简介：全平台剧集实时热度监控与数据分析工具
   - 小程序头像：上传一个logo（可用Canva等工具设计）
   - 服务类目选择：「工具」→「信息查询」
     或「生活服务」→「生活服务」
```

**步骤3：获取AppID和AppSecret**

```
1. 左侧菜单「开发管理」→「开发设置」
2. 记录下：
   - AppID(小程序ID)：wx1234567890abcdef
   - AppSecret(小程序密钥)：点击「重置」获取，妥善保存
```

**步骤4：配置服务器域名**

```
1. 「开发管理」→「开发设置」→「服务器域名」
2. 点击「修改」（需要管理员扫码）
3. 配置：
   - request合法域名：https://你的域名
   - uploadFile合法域名：https://你的域名
   - downloadFile合法域名：https://你的域名

注意：
- 域名必须是已备案的
- 必须是https协议
- 不能使用IP地址
- 不能使用localhost
```

**步骤5：下载微信开发者工具**

```
1. 访问 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 下载对应系统版本（Windows/Mac）
3. 安装并启动
4. 用微信扫码登录
5. 创建新项目：
   - 项目名称：drama-tracker
   - 目录：选择一个本地文件夹
   - AppID：填入步骤3获取的AppID
   - 后端服务：不使用云服务
   - 语言：JavaScript
6. 点击「确定」创建项目
```

## 第十一部分：域名备案与服务器部署

### 11.1 域名备案流程

```
前提：微信小程序的服务器域名必须已完成ICP备案

备案流程（以腾讯云为例）：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

步骤1：域名实名认证
- 如果域名不在腾讯云，需要先将DNS解析转到腾讯云
  或在域名注册商处完成实名认证
- 域名实名认证需要3个工作日

步骤2：购买腾讯云服务器
- 备案要求服务器在对应云服务商处
- 购买后获得备案服务码

步骤3：腾讯云备案
1. 登录腾讯云控制台
2. 进入「网站备案」
3. 开始备案 → 填写信息：
   - 备案类型：首次备案
   - 主体信息：
     * 主办单位性质：个人
     * 姓名、身份证号、手机号
     * 通信地址
   - 网站信息：
     * 域名：你的域名
     * 网站名称：（不能含"中国""国家"等词）
     * 网站用途描述
     * 网站服务内容：其他
4. 上传资料：
   - 身份证正反面照片
   - 手持身份证照片
   - 备案真实性核验照片
5. 提交初审（腾讯云审核，1-2个工作日）
6. 初审通过后提交管局审核
7. 短信核验（工信部发短信，24小时内验证）
8. 管局审核（各省不同，一般7-20个工作日）
9. 备案通过，获得ICP备案号

总耗时预估：15-30天
建议：尽早开始备案，同时进行开发工作

步骤4：备案后配置
- 在网站底部添加备案号
- 在小程序管理后台配置服务器域名
```

## 第十七部分：常见问题与解决方案

**Q1：我完全不懂代码，真的能做出来吗？**

A1：可以。本方案设计为让神奇奶酪逐步生成所有代码，你只需要：按步骤准备好环境、按顺序向神奇奶酪请求代码、将代码复制到对应文件中、按照说明运行。实际上需要你做的主要是"复制粘贴"和"执行命令"。

**Q2：平台改版导致采集失败怎么办？**

A2：这是最常见的问题。解决方案：采集系统加入异常监控，失败时邮件/微信通知你；失败后把新页面的HTML发给神奇奶酪，让它帮你更新采集代码；预留手动录入数据的后台接口。

**Q3：数据准确性如何保证？**

A3：实时热度直接读取平台公开数据，与平台一致；播放量通过差值计算，可能有微小误差；加入异常检测机制，自动过滤异常数据；数据说明页透明公开计算方法。

**Q4：服务器被封/IP被屏蔽怎么办？**

A4：控制采集频率，不要太快；使用代理IP池；增加随机延迟；模拟正常浏览器行为。

**Q5：小程序审核不通过怎么办？**

A5：常见被拒原因和解决："内容不完整"→确保所有页面有内容；"类目不符"→调整服务类目；"涉嫌抄袭"→确保UI是原创的；"数据来源不明"→添加数据说明页面。

**Q6：个人主体小程序有什么限制？**

A6：不能开通微信支付；不能使用某些敏感API；不影响本项目的核心功能；如需要，后续可注册个体工商户升级。

**Q7：用户量大了怎么办？**

A7：升级服务器配置；数据库独立部署；增加Redis缓存覆盖；考虑CDN加速；考虑负载均衡。

**Q8：备案期间能开发吗？**

A8：完全可以！备案需要15-30天，在开发者工具中可以不用真实域名，勾选"不校验合法域名"选项，备案期间完成开发，备案后直接部署。

**Q9：一个人能维护吗？**

A9：初期完全可以。日常维护工作：每天查看采集任务状态（5分钟）；新剧上线时录入数据（每周30分钟）；采集器异常时修复（不定期）；建议写一个简单的管理后台来辅助。

**Q10：如何保证数据和云合/酷云的差异？**

A10：我们读取平台公开数据，不是估算数据；云合、酷云是"有效播放"估算模型；我们直接用平台自有热度值，更透明；我们的"剧力指数"是独立体系，是差异化；明确告知用户我们的数据口径和来源。
