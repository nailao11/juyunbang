# 剧云榜 完整部署指南

> **服务器配置**：4核4GB内存 + 40GB云硬盘
>
> **项目功能**：实时采集爱奇艺/腾讯视频/优酷/芒果TV四大平台的站内热度值，通过微信小程序展示
>
> **采集原理**：Playwright 无头浏览器渲染页面后提取热度值（纯HTTP请求无法获取JS渲染的数据）
>
> **采集策略**：只采集近30天内上线的新剧，每15分钟增量采集，同剧同平台10分钟内不重复

## 前置准备

开始之前，你需要准备好以下信息：

| 项目 | 说明 | 在哪获取 |
|---|---|---|
| 域名 | 如 sqnl8.cn | 你的域名服务商 |
| 微信小程序 AppID | 小程序唯一标识 | mp.weixin.qq.com → 开发管理 → 开发设置 |
| 微信小程序 AppSecret | 小程序密钥 | 同上 |
| 七牛云 AccessKey | 对象存储密钥 | portal.qiniu.com → 密钥管理 |
| 七牛云 SecretKey | 对象存储密钥 | 同上 |
| 七牛云 Bucket 名称 | 存储空间名 | portal.qiniu.com → 对象存储 → 新建空间 |
| 七牛云域名 | CDN域名 | 创建Bucket后绑定的域名 |

### 七牛云是什么？为什么需要？

七牛云用于存储用户上传的图片（头像、反馈截图等）。小程序无法直接把图片存在你的服务器上，需要一个对象存储服务来托管图片文件。

**如果暂时不需要头像和图片功能**，可以先跳过七牛云配置，.env 中留空即可，不影响核心的热度采集和展示功能。

## 第二步：配置 Swap

> Playwright 启动 Chromium 浏览器单实例占用 300-500MB 内存。
> 4GB 内存同时运行 Flask + MariaDB + Redis + Chromium，峰值可能超 4GB。
> 加 2GB Swap 防止内存不足时进程被系统强制杀掉（OOM Kill）。

```bash
# 检查是否已有 Swap
swapon --show
```

**如果没有任何输出**，执行以下命令创建：
```bash
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

# 写入开机自动挂载
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

**如果已有输出**，跳过此步。

验证：
```bash
free -h
# Swap 行应显示约 2.0G
```

- Set root password? **Y**，然后输入你想设的root密码
- Remove anonymous users? **Y**
- Disallow root login remotely? **Y**
- Remove test database? **Y**
- Reload privilege tables? **Y**

然后创建项目数据库和用户：

```sql
CREATE USER 'juyunbang'@'localhost' IDENTIFIED BY '你的数据库密码';
GRANT ALL PRIVILEGES ON juyunbang.* TO 'juyunbang'@'localhost';
```

导入表结构和初始数据：

## 第十三步：微信小程序配置

### 13.1 微信公众平台配置

登录 [微信公众平台](https://mp.weixin.qq.com)：

1. **开发管理** → **开发设置** → **服务器域名**：
   - request 合法域名添加：`https://api.sqnl8.cn`
   - uploadFile 合法域名添加：`https://api.sqnl8.cn`

2. 如果配置了七牛云，还需添加：
   - downloadFile 合法域名添加：`https://你的七牛云CDN域名`

### 13.2 微信开发者工具配置

在**电脑上**（不是服务器上）操作：

1. 下载并安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)

2. 把 `miniapp/` 目录复制到电脑上（或直接在电脑上 clone 仓库）

3. 打开微信开发者工具 → **导入项目** → 选择 `miniapp` 目录

4. **修改 AppID**：
   编辑 `miniapp/project.config.json`，找到：
   ```json
   "appid": "wxTODO_REPLACE_WITH_REAL_APPID"
   ```
   替换为你的真实小程序 AppID

5. **修改 API 地址**：
   编辑 `miniapp/app.js`，确认 `globalData.baseUrl` 为：
   ```javascript
   baseUrl: 'https://api.sqnl8.cn/api/v1'
   ```

6. **预览测试**：点击工具栏"预览"按钮，用手机微信扫码测试

7. **上传发布**：确认功能正常后，点击"上传" → 在微信公众平台提交审核

## 定时任务说明

采集服务 (`juyunbang-crawler`) 启动后自动运行以下定时任务：

| 时间 | 任务 | 说明 |
|---|---|---|
| 每15分钟 | 热度采集 | Playwright 打开四平台页面提取热度值 |
| 每日 00:00 | 数据清洗 | 去除异常值和重复记录 |
| 每日 00:30 | 日度统计 | 计算每日热度均值/最高/最低 |
| 每日 01:00 | 剧力指数 | 综合热度+播放+社交+口碑计算指数 |
| 每日 01:30 | 异动检测 | 检测热度暴涨的剧（>200%） |
| 每日 03:00 | 豆瓣评分 | 更新在播剧的豆瓣评分 |
| 每日 04:00 | 旧数据清理 | 删除30天前的实时热度和播放量快照 |
| 每日 15:00 | 日度数据发布 | 把前一日已计算完成的 heat_daily / playcount_daily / drama_index_daily 标记为 published |

## 常见问题

**Q: Playwright 安装报错？**
```bash
cd /opt/juyunbang/backend
./venv/bin/pip uninstall -y playwright
./venv/bin/pip install playwright==1.40.0
./venv/bin/playwright install chromium --force
./venv/bin/playwright install-deps chromium
```

**Q: 采集服务启动后没有数据？**
1. 查看日志：`tail -50 /opt/juyunbang/logs/scheduler_$(date +%Y-%m-%d).log`
2. 确认 dramas 表中有近30天 air_date 的数据：
   ```bash
   mysql -u juyunbang -p juyunbang -e "SELECT title, air_date, status FROM dramas WHERE air_date > DATE_SUB(NOW(), INTERVAL 30 DAY);"
   ```
3. 手动测试：`cd /opt/juyunbang/backend && ./venv/bin/python test_crawl.py --platform tencent`

**Q: API 返回 502 Bad Gateway？**
```bash
systemctl status juyunbang-api        # 看是否在运行
journalctl -u juyunbang-api -n 50     # 看错误日志
systemctl restart juyunbang-api       # 重启
```

**Q: 内存不足进程被杀？**
```bash
free -h                                # 确认Swap已启用
# 如果 Swap 行全是 0，回到第二步创建Swap
# 查看是否有进程被 OOM Kill：
dmesg | grep -i "out of memory" | tail -5
```

**Q: SSH 断开重连后命令报错？**
```bash
# 安卓SSH重连后自动回到 /root，需要重新切目录
cd /opt/juyunbang/backend
# 如果需要执行 pip/playwright 等命令
source venv/bin/activate
```

**Q: MariaDB 连接被拒绝？**
```bash
systemctl status mariadb              # 确认在运行
systemctl start mariadb               # 如果没运行就启动
# 检查 .env 中的 DB_PASSWORD 是否与第六步设置的一致
```

**Q: Nginx 报 SSL 证书错误？**
```bash
# 检查证书文件是否存在
ls /etc/letsencrypt/live/sqnl8.cn/
# 如果不存在，回到第九步申请证书
# 如果域名不同，编辑 Nginx 配置：
vi /opt/juyunbang/deploy/nginx.conf
# 修改 server_name 和 ssl_certificate 路径
cp /opt/juyunbang/deploy/nginx.conf /etc/nginx/sites-available/juyunbang
```
