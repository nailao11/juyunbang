# 热剧榜 / 热剧助手 新手友好部署指南（优化重写版）

> 这是一份 **给新手** 的部署指南。  
> 它的目标不是“讲原理”，而是让你 **尽量少出错地一步一步部署成功**。  
> 我会把每一步需要在哪个目录执行、哪些内容必须改、哪些内容不用改，都写清楚。

---

## 一、这份文档解决什么问题

原来的部署指南已经有很多有用信息，但对新手来说有几个明显问题：

1. **没有反复强调当前命令应该在哪个目录执行**  
   SSH 断线重连后，目录经常会回到 `/root`，如果你不先切回正确目录，就很容易执行错地方。

2. **用了 `vi .env` 这种对新手不友好的操作方式**  
   很多人并不熟悉 `vi`。  
   本文档改成：**要么直接复制整段命令自动生成 `.env`，要么使用 `nano` 编辑器**。

3. **有些值到底要不要自己填、要不要记住，不够清楚**  
   本文档会把配置项分成两类：
   - **必须由你自己替换的内容**
   - **自动生成即可、不需要你记住的内容**

4. **有些步骤混在一起，看不出前后依赖**
   本文档会把部署拆成一个很清晰的顺序：
   - 准备
   - 连接服务器
   - 安装系统环境
   - 拉代码
   - 配 `.env`
   - 数据库初始化
   - SSL
   - Nginx
   - systemd 服务
   - 验证
   - 小程序配置

---

## 二、先说最重要的规则：你要时刻知道自己“现在在哪个目录”

### 1）以后每次 SSH 重新连接后，先执行这两个命令：

```bash
pwd
ls
```

### 2）如果你现在不确定自己在哪个目录，直接看 `pwd` 的输出

常见目录说明：

- `/root`：root 用户的家目录。**SSH 重连后经常会回到这里**
- `/opt`：我们准备放项目代码的地方
- `/opt/rejubang`：项目根目录
- `/opt/rejubang/backend`：后端目录
- `/opt/rejubang/miniapp`：小程序目录
- `/etc/nginx/sites-available`：Nginx 配置目录
- `/etc/systemd/system`：systemd 服务目录

### 3）本文档中每个步骤都会写明“当前目录应该是什么”

你一定要养成习惯：

- **看到命令前，先确认当前目录**
- **不确定就先 `pwd`**
- **SSH 断开重连后，默认重新从 `/root` 开始**

## 三、你在开始前必须准备好的信息

请先准备下面这些内容。没有这些内容，不要急着往下做。

| 项目 | 是否必须 | 说明 |
|---|---|---|
| 服务器公网 IP | 必须 | 用来 SSH 登录服务器 |
| root 密码（或 SSH 密钥） | 必须 | 用来登录服务器 |
| 域名根域名 | 必须 | 例如 `nailao.asia` |
| API 子域名 | 必须 | 例如 `api.nailao.asia` |
| 微信小程序 AppID | 必须 | 在微信公众平台后台获取 |
| 微信小程序 AppSecret | 必须 | 在微信公众平台后台获取 |
| 数据库密码（你自己设置一个） | 必须 | 例如你自己想一个强密码 |
| 七牛云配置 | 可选 | 如果你暂时不做图片上传，可以先留空 |

## 四、这份指南默认的部署目标

本项目当前代码与部署文件体现出来的默认部署方式是：

- 后端目录：`/opt/rejubang/backend`
- Flask / Gunicorn 监听：`127.0.0.1:5000`

- Redis：本机

- API 服务名：`rejubang-api`

## 五、操作约定（非常重要）

### 1）除非我特别写明“在你自己的电脑上操作”，否则默认都是：

> **在服务器终端中执行命令**

### 2）默认你使用的是 `root` 登录服务器

如果你不是 `root`，需要自己在命令前加 `sudo`，但为了减少混乱，这份文档先按 `root` 写。

### 3）编辑文件时，优先使用 `nano`，不使用 `vi`

如果系统里没有 `nano`，本文档会先安装它。

### 4）自动生成的随机密钥，不需要你记住

像下面这些配置：

- `SECRET_KEY`
- `JWT_SECRET_KEY`

它们应该是 **每台服务器唯一** 的。  
为了安全，我**不会**在文档里写死一组公共固定值。  
但我会让命令 **自动生成并写入 `.env`**，这样你既不用记，也不用手工乱填。

## 第 3 步：配置 Swap（防止内存不够时进程被杀）

### 当前目录
> 任意目录都可以

先检查有没有 Swap：

```bash
swapon --show
```

### 情况 A：如果什么都没显示
说明还没有 Swap，就执行下面这些命令：

```bash
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

### 情况 B：如果已经有输出
说明已经启用了 Swap，这一步跳过。

### 验证

```bash
free -h
```

你会看到一行 `Swap`，大约是 `2.0G` 左右。

## 第 7 步：启动并配置 MariaDB 数据库

### 当前目录
> 任意目录都可以

### 启动数据库并设置开机自启

```bash
systemctl start mariadb
systemctl enable mariadb
```

### 运行安全初始化

```bash
mysql_secure_installation
```

### 新手建议这样回答

它会问你多个问题，建议按下面来：

- `Enter current password for root:`  
  **第一次安装一般直接回车**

- `Switch to unix_socket authentication?`  
  **N**

- `Change the root password?`  
  **Y**

然后输入你给 MariaDB root 设置的新密码。

- `Remove anonymous users?`  
  **Y**

- `Disallow root login remotely?`  
  **Y**

- `Remove test database and access to it?`  
  **Y**

- `Reload privilege tables now?`  
  **Y**

### 登录数据库

```bash
mysql -u root -p
```

输入你刚刚设置的 root 数据库密码。

### 在 MariaDB 里执行这些 SQL

> 注意：下面是在 `mysql>` 提示符里输入，不是在 Linux 终端里输入。

```sql
CREATE DATABASE rejubang DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'rejubang'@'localhost' IDENTIFIED BY '你自己设置的数据库密码';
GRANT ALL PRIVILEGES ON rejubang.* TO 'rejubang'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 这里有两个不同的密码，不要混淆

1. **MariaDB root 密码**  
   这是你运行 `mysql_secure_installation` 时设置的

2. **项目数据库用户 `rejubang` 的密码**  
   这是你执行 `CREATE USER 'rejubang'@'localhost' IDENTIFIED BY '这里';` 时设置的

后面 `.env` 里要写的是 **第二个密码**，不是 root 密码。

## 第 9 步：生成并写入 `.env`（重点优化版，不用 vi）

这一节是整份文档里最关键、也最容易让新手混乱的部分。

### 当前目录
> **必须在 `/opt/rejubang/backend`**

先确认：

```bash

pwd
```

应输出：

```bash
/opt/rejubang/backend
```

### 第 9.1 步：先决定你要填写哪些真实信息

你只需要准备下面这些你自己真实的值：

- 你的小程序 AppID
- 你的小程序 AppSecret
- 你给项目数据库用户 `rejubang` 设置的密码
- 七牛云配置（可选，不用可以先留空）

---

### 第 9.2 步：把下面这整段命令复制到服务器执行

> **这一段命令会自动生成 `SECRET_KEY` 和 `JWT_SECRET_KEY`**  
> 你不用自己记，也不用手工输随机字符。

```bash
cd /opt/rejubang/backend

SECRET_KEY=$(python3 -c "import secrets; print(secrets.token_hex(32))")
JWT_SECRET_KEY=$(python3 -c "import secrets; print(secrets.token_hex(32))")

cat > /opt/rejubang/backend/.env <<'EOF'
FLASK_ENV=production
SECRET_KEY=__SECRET_KEY_PLACEHOLDER__
DB_HOST=localhost
DB_PORT=3306
DB_NAME=rejubang
DB_USER=rejubang
DB_PASSWORD=__DB_PASSWORD_PLACEHOLDER__
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0
WX_APPID=__WX_APPID_PLACEHOLDER__
WX_SECRET=__WX_SECRET_PLACEHOLDER__
QINIU_ACCESS_KEY=
QINIU_SECRET_KEY=
QINIU_BUCKET=rejubang
QINIU_DOMAIN=
JWT_SECRET_KEY=__JWT_SECRET_KEY_PLACEHOLDER__
JWT_ACCESS_TOKEN_EXPIRES=86400
EOF

sed -i "s/__SECRET_KEY_PLACEHOLDER__/$SECRET_KEY/g" /opt/rejubang/backend/.env
sed -i "s/__JWT_SECRET_KEY_PLACEHOLDER__/$JWT_SECRET_KEY/g" /opt/rejubang/backend/.env
```

执行完成后，`.env` 文件已经生成好了。

---

### 第 9.3 步：现在只改“必须改”的几项

不要用 `vi`，改用 `nano`：

```bash
nano /opt/rejubang/backend/.env
```

### 你只需要修改下面这几项

```ini
DB_PASSWORD=__DB_PASSWORD_PLACEHOLDER__
WX_APPID=__WX_APPID_PLACEHOLDER__
WX_SECRET=__WX_SECRET_PLACEHOLDER__
```

如果你要启用七牛云图片上传，再改这几项：

```ini
QINIU_ACCESS_KEY=
QINIU_SECRET_KEY=
QINIU_DOMAIN=
```

### 如果你暂时不用七牛云
就保持它们为空，不影响核心热度采集和 API 功能。

---

### 第 9.4 步：`nano` 怎么保存退出

在 `nano` 里修改完以后：

1. 按 `Ctrl + O`  
   这是保存（Write Out）

2. 它会提示文件名，直接按回车

3. 再按 `Ctrl + X`  
   这是退出

---

### 第 9.5 步：检查 `.env` 是否写对了

```bash
cat /opt/rejubang/backend/.env
```

你要重点确认：

- `DB_PASSWORD` 已经不是占位符了
- `WX_APPID` 已经是真实 AppID
- `WX_SECRET` 已经是真实 AppSecret
- `SECRET_KEY` 和 `JWT_SECRET_KEY` 已经有一长串随机字符

### 重要说明

- `SECRET_KEY` 和 `JWT_SECRET_KEY` **不需要你手动修改**
- 也**不需要你额外记住**
- 只要 `.env` 文件保留在服务器上，项目就会一直用它

## 第 14 步：配置 Nginx（新手友好版）

---

### 第 14.1 步：如果你使用的就是仓库默认域名 `nailao.asia`

直接执行：

```bash
cp /opt/rejubang/deploy/nginx.conf /etc/nginx/sites-available/rejubang
```

### 第 14.2 步：如果你使用的是自己的域名（不是 `nailao.asia`）

你必须先改配置文件，再复制到 Nginx 目录。

#### 先复制一份到系统目录

```bash
cp /opt/rejubang/deploy/nginx.conf /etc/nginx/sites-available/rejubang
```

#### 用 `nano` 打开

```bash
nano /etc/nginx/sites-available/rejubang
```

#### 你要修改的内容主要有三处

1. `server_name` 里的域名  
2. `ssl_certificate` 路径里的域名  
3. `ssl_certificate_key` 路径里的域名  

例如原文件里有这些内容：

```nginx
server_name nailao.asia api.nailao.asia;
server_name api.nailao.asia;
ssl_certificate /etc/letsencrypt/live/nailao.asia/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/nailao.asia/privkey.pem;
```

如果你的域名是 `example.com` 和 `api.example.com`，就改成：

```nginx
server_name example.com api.example.com;
server_name api.example.com;
ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;
```

#### 保存并退出
- `Ctrl + O`
- 回车
- `Ctrl + X`

#### 启用配置

```bash
ln -sf /etc/nginx/sites-available/rejubang /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl restart nginx
```

## 第 24 步：最常见的错误和解决办法

### 问题 1：SSH 重连后命令突然报错
**原因：** 你回到了 `/root`，但你以为自己还在 `/opt/rejubang/backend`

**解决：**

```bash
pwd
cd /opt/rejubang/backend
pwd
```

---

### 问题 2：`.env` 配好了，但服务启动失败
**先看日志：**

```bash
journalctl -u rejubang-api -n 50
```

**最常见原因：**
- `DB_PASSWORD` 写错了
- `WX_APPID` 或 `WX_SECRET` 还是占位符
- `.env` 没有保存在 `/opt/rejubang/backend/.env`
- `SECRET_KEY` 和 `JWT_SECRET_KEY` 没生成成功

**检查：**

```bash
cat /opt/rejubang/backend/.env
```

---

### 问题 3：Nginx `nginx -t` 报错
**最常见原因：**
- SSL 证书路径写错
- 你用的是自己的域名，但没改 `server_name`
- 证书目录名和域名不一致

**检查证书目录：**

```bash
ls /etc/letsencrypt/live/
```

看清楚你的证书目录名到底是什么。

---

### 问题 4：API 返回 502 Bad Gateway
说明 Nginx 能工作，但后端没正常提供服务。

**先检查后端服务状态：**

```bash
systemctl status rejubang-api
journalctl -u rejubang-api -n 50
```

再检查本机健康检查：

```bash
curl http://127.0.0.1:5000/health
```

---

### 问题 5：采集服务启动了，但数据库没数据
先看采集日志：

```bash
journalctl -u rejubang-crawler -n 100
tail -f /opt/rejubang/logs/scheduler_$(date +%Y-%m-%d).log
```

再手动测试：

```bash
cd /opt/rejubang/backend
./venv/bin/python test_crawl.py --platform tencent
```

---

### 问题 6：不知道自己该在哪里执行命令
记住最简单的处理方式：

```bash
pwd
```

如果你发现自己不在该在的目录，就重新切过去。

最常用的三个目录：

```bash
cd /root
cd /opt/rejubang
cd /opt/rejubang/backend
```

---

## 第 25 步：给新手的最后建议

1. **每执行一大步前先 `pwd`**
2. **不要同时开多个 SSH 窗口乱操作**
3. **不要一看到报错就乱重装**
4. **先看日志，再处理**
5. **`.env` 不要随便删**
6. **数据库密码和 `.env` 里的密码要保持一致**
7. **域名不是 `nailao.asia` 的话，一定要改 Nginx 里的域名和证书路径**
8. **随机密钥不需要记住，只要 `.env` 文件在就行**
