## 1. 服务状态速查

### 1.1 一条命令看全部关键服务

```
root@server:~# systemctl is-active mariadb redis-server nginx rejubang-api rejubang-crawler
active
active
active
active
active
```

**五行全 active** 就是健康状态。哪个不是 active，对应的处理见下。

### 1.2 按服务看详细状态

```
root@server:~# systemctl status rejubang-api --no-pager
root@server:~# systemctl status rejubang-crawler --no-pager
root@server:~# systemctl status mariadb --no-pager
root@server:~# systemctl status redis-server --no-pager
root@server:~# systemctl status nginx --no-pager
```

### 1.3 系统资源

```
root@server:~# free -h          # 内存 + Swap 使用
root@server:~# df -h             # 磁盘使用
root@server:~# uptime            # 启动时间 + 负载
```

重点关注：
- `free -h` 的 `available` 不低于 500Mi，`Swap used` 正常不超过 1G
- `df -h` 中 `/` 根分区使用率 < 85%

## 2. 重启服务

### 2.1 重启 API 服务（改了代码 / `.env` 后必重启）

```
root@server:~# systemctl restart rejubang-api
```

### 2.2 重启爬虫服务（改了爬虫代码 / 调度逻辑后必重启）

```
root@server:~# systemctl restart rejubang-crawler
```

### 2.3 重启 Nginx（改了 nginx.conf 后必重启）

```
root@server:~# nginx -t && systemctl restart nginx
```

**先 `nginx -t` 检查语法**，语法没问题再 restart，避免配置错误导致 Nginx 直接起不来。

### 2.4 一次重启所有核心服务

```
root@server:~# systemctl restart rejubang-api rejubang-crawler nginx
```

## 3. 查日志

### 3.1 API 服务日志（systemd 管理，用 journalctl）

```
# 实时跟踪日志（Ctrl+C 退出）
root@server:~# journalctl -u rejubang-api -f

# 最近 100 行
root@server:~# journalctl -u rejubang-api -n 100 --no-pager

# 今天的全部
root@server:~# journalctl -u rejubang-api --since today --no-pager

# 只看错误
root@server:~# journalctl -u rejubang-api -p err --since today --no-pager
```

### 3.2 爬虫服务日志

```
# systemd 侧的日志
root@server:~# journalctl -u rejubang-crawler -f

# 项目自己写的详细日志（按天分文件）
root@server:~# tail -f /opt/rejubang/logs/scheduler_$(date +%Y-%m-%d).log

# 看昨天的
root@server:~# tail -200 /opt/rejubang/logs/scheduler_$(date -d yesterday +%Y-%m-%d).log
```

### 3.3 Nginx 访问和错误日志

```
# 所有请求的访问日志
root@server:~# tail -f /var/log/nginx/access.log

# 错误日志（502 / 504 / 连接错误会写在这里）
root@server:~# tail -f /var/log/nginx/error.log
```

### 3.4 MariaDB 错误日志

```
root@server:~# tail -100 /var/log/mysql/error.log
```

## 5. 查数据库里的数据

### 5.1 最新 20 条热度记录

```
root@server:~# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang -e "
SELECT d.title, p.name AS platform, h.heat_value, h.record_time
FROM heat_realtime h
JOIN dramas d ON h.drama_id = d.id
JOIN platforms p ON h.platform_id = p.id
ORDER BY h.record_time DESC LIMIT 20;
"
```

### 5.2 各平台近 30 天剧数量

```
root@server:~# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang -e "
SELECT p.name, COUNT(DISTINCT dp.drama_id) AS cnt
FROM platforms p
LEFT JOIN drama_platforms dp ON dp.platform_id = p.id
LEFT JOIN dramas d ON d.id = dp.drama_id
WHERE d.air_date > DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY p.id;
"
```

### 5.3 今日采集任务执行情况

```
root@server:~# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang -e "
SELECT task_name, status, message, started_at, finished_at
FROM crawl_tasks
WHERE DATE(started_at) = CURDATE()
ORDER BY started_at DESC LIMIT 20;
"
```

## 7. 数据库备份

### 7.1 建立备份目录

```
root@server:~# mkdir -p /opt/backup
```

### 7.2 手动备份一次

```
root@server:~# mysqldump -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' \
  --default-character-set=utf8mb4 \
  --single-transaction \
  rejubang > /opt/backup/rejubang_$(date +%Y%m%d_%H%M%S).sql

root@server:~# ls -lh /opt/backup/
-rw-r--r-- 1 root root 500K Apr 21 15:30 rejubang_20260421_153012.sql
```

### 7.3 设置自动备份（每天凌晨 3:30，保留 14 天）

```
root@server:~# crontab -e
```

第一次打开会让你选编辑器，**选 `nano`**（选 `1` 回车）。

在文件末尾追加一行：

```
30 3 * * * mysqldump -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' --single-transaction --default-character-set=utf8mb4 rejubang > /opt/backup/rejubang_$(date +\%Y\%m\%d).sql && find /opt/backup/ -name "rejubang_*.sql" -mtime +14 -delete
```

**nano 保存**：`Ctrl + O` → 回车 → `Ctrl + X`。

> ⚠️ **crontab 里的 `%` 必须写成 `\%`** 才会被正确转义成 `%`。

验证已添加：

```
root@server:~# crontab -l
```

### 8.1 列出已有备份

```

```

### 8.2 恢复指定备份（**会覆盖现有数据**）

```
root@server:~# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang < /opt/backup/rejubang_20260421.sql
```

恢复前建议**先做一次当前备份**以防万一：

```
root@server:~# mysqldump -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' --single-transaction rejubang > /opt/backup/before_restore_$(date +%Y%m%d_%H%M%S).sql
```

## 9. 迁移到新服务器

### 9.1 在**旧服务器**上打包

```
root@old-server:~# mysqldump -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' --single-transaction --default-character-set=utf8mb4 rejubang > /tmp/rejubang.sql
```

同时把旧服务器的 `.env` 也带过去（里面有小程序 AppSecret 和七牛云密钥）：

```
root@old-server:~# cp /opt/rejubang/backend/.env /tmp/rejubang.env
```

### 9.2 从旧服务器传到新服务器

在**旧服务器**上 scp 到新机器：

```
root@old-server:~# scp /tmp/rejubang.sql /tmp/rejubang.env root@<新服务器IP>:/tmp/
```

### 9.3 新服务器走完第二~六册

在**新服务器**上按 [02-系统环境安装.md](./02-系统环境安装.md) 到 [06-SSL证书与Nginx.md](./06-SSL证书与Nginx.md) 完整执行一遍，但到 [05-环境变量配置.md](./05-环境变量配置.md) 时：

**不要重新生成 `.env`，直接用旧的**：

```
root@new-server:~# cp /tmp/rejubang.env /opt/rejubang/backend/.env
root@new-server:~# chmod 600 /opt/rejubang/backend/.env
```

建库建用户后直接导入备份：

```
root@new-server:~# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang < /tmp/rejubang.sql
```

然后第七册启动服务。

### 9.4 最后一步：DNS 换指向

在域名服务商控制台把 A 记录指向**新服务器 IP**。

DNS 生效后（5~30 分钟）再关闭旧服务器。期间可能有少量 5xx，属于预期。

## 12. 清理磁盘空间

### 12.0 项目的自动归档策略（2026-04 加入）

爬虫服务每天 04:00 自动运行 `job_archive_old_heat`：

- `heat_realtime`（原始热度）：**保留 90 天**，超出自动删除（已先被日聚合保留在 `heat_daily`）
- `playcount_snapshot`（播放量快照）：**保留 90 天**
- `heat_daily`（每天一条的聚合数据）：**保留 365 天**，支持小程序查一年趋势图

预期增速：
- `heat_realtime`：~1 MB/天 × 90 天 ≈ 90 MB（稳态）
- `heat_daily`：365 天 × 30 剧 × 4 平台 ≈ 50 KB（稳态）

**正常情况下不需要手动清理数据库**。

### 12.1 看是什么占了空间

```
root@server:~# du -sh /opt/rejubang/logs/
root@server:~# du -sh /var/log/
root@server:~# du -sh /opt/backup/
```

### 12.2 清理 30 天前的项目日志

```
root@server:~# find /opt/rejubang/logs/ -name "*.log" -mtime +30 -delete
```

### 12.3 清理 journalctl 日志（保留最近 7 天）

```
root@server:~# journalctl --vacuum-time=7d
```

### 12.4 清理 apt 缓存

```
root@server:~# apt autoremove -y && apt clean
```
