# 第七册：启动服务、验证、录入在播剧

> **本册在哪操作**：**云服务器** SSH 命令行（启动服务）+ **本地浏览器**（访问管理后台）。
>
> **目标**：把 API 和爬虫服务起起来，然后**通过管理后台录入在播剧**（核心步骤），验证整条链路通畅。

---

## 1. 两个后端服务是什么

| systemd 服务名 | 作用 | 运行什么 | 端口 |
|---|---|---|---|
| `rejubang-api` | Flask API，处理小程序请求 + 提供管理后台 | `gunicorn run:app` | 127.0.0.1:5000 |
| `rejubang-crawler` | 爬虫调度器，每 15 分钟对已录入的剧采热度 | `python scheduler/task_scheduler.py` | 不监听端口 |

两者共用 `/opt/rejubang/backend/venv/` 虚拟环境和 `/opt/rejubang/backend/.env` 配置。

---

## 2. 创建日志目录

```
root@server:~# mkdir -p /opt/rejubang/logs
```

---

## 3. 安装 systemd 服务文件

```
root@server:~# cd /opt/rejubang/deploy
root@server:/opt/rejubang/deploy# cp rejubang-api.service /etc/systemd/system/
root@server:/opt/rejubang/deploy# cp rejubang-crawler.service /etc/systemd/system/
root@server:/opt/rejubang/deploy# systemctl daemon-reload
```

---

## 4. 启动两个服务

```
root@server:~# systemctl enable --now rejubang-api
root@server:~# systemctl enable --now rejubang-crawler
```

### 4.1 状态检查

```
root@server:~# systemctl is-active rejubang-api rejubang-crawler
active
active
```

两行 `active` 就对了。若有 `failed` 看日志：

```
root@server:~# journalctl -u rejubang-api -n 100 --no-pager
root@server:~# journalctl -u rejubang-crawler -n 100 --no-pager
```

## 5. API 健康检查

### 5.1 本机直连后端

```
root@server:~# curl http://127.0.0.1:5000/health
{"status":"ok","service":"rejubang-api"}
```

### 5.2 外网 HTTPS

```
root@server:~# curl https://<你的API域名>/health
{"status":"ok","service":"rejubang-api"}
```

在**本地浏览器**访问 `https://<你的API域名>/api/v1/heat/realtime/rank`，应返回 JSON（`data` 字段此时是空数组，因为还没有数据）。

## 6. 【关键新步骤】通过管理后台录入在播剧

> ⚠️ **本项目架构变更说明（2026-04）**：不再自动"发现"新剧，**由你手动录入在播剧清单**。工作量很小——每月新剧也就 5~10 部，录一部只要 1 分钟。
>
> **如果跳过这一步，爬虫会正常运行但没有任何剧可采，数据库永远是空的。**

### 6.1 打开管理后台

**本地电脑**浏览器访问：

```
https://<你的API域名>/admin
```

（例如 `https://api.nailao.asia/admin`）

会看到一个登录页，要求输入 Admin Token。

### 6.2 用 `ADMIN_TOKEN` 登录

- 粘贴第五册 `.env` 里的 `ADMIN_TOKEN`（默认预设值是 `zCfQR0OkgWbZwaG4iw5Vb1E3ksvGAxTpR8nGk7gd--c`）
- 点"登录"
- 成功后进入管理面板

### 6.3 录入一部剧（以"蜜语纪"为例）

在"➕ 新增 / 更新剧集"表单里填：

| 字段 | 值 |
|---|---|
| 剧名 | `蜜语纪` |
| 开播日期 | `2026-04-13` |
| 类型 | 电视剧 |
| 腾讯视频 | `https://v.qq.com/x/cover/mzc002006dzzunf.html` <br>（或只填 `mzc002006dzzunf` 也可） |
| 爱奇艺 | `https://www.iqiyi.com/v_pz64qf5dtk.html` <br>（或只填 `v_pz64qf5dtk`） |
| 优酷 / 芒果TV | 没有就留空 |

点击 **"先测试热度提取"** → 浏览器显示每个平台的实时抓取结果（约 10~30 秒）：
```
正在测试 tencent …
✓ tencent 热度/播放量 = 24096
正在测试 iqiyi …
✓ iqiyi 热度/播放量 = 17868
```

看到值 > 0 说明链路通畅。然后点 **"保存录入"**。

### 6.4 一键导入样例数据（用于快速验证）

如果你想一次导入多部在播剧测试，不手动一条条录，可以直接跑样例 SQL：

```
root@server:~# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang \
  < /opt/rejubang/backend/migrations/test_dramas.sql
```

会导入 3 部测试剧（蜜语纪、方圆八百米、优酷测试剧），覆盖腾讯/爱奇艺/优酷三个平台。导入后在 `/admin` 页面底部"当前在播剧清单"能看到它们。

### 6.5 哪里找各平台的 URL？

| 平台 | 怎么拿 |
|---|---|
| 腾讯视频 | 浏览器搜索剧名 → 点进详情页 → 复制地址栏的 URL |
| 爱奇艺 | 同上。URL 形如 `www.iqiyi.com/v_xxxxxxx.html` |
| 优酷 | 同上。URL 形如 `v.youku.com/v_show/id_xxxxxxxx.html` |
| 芒果TV | 同上。URL 形如 `www.mgtv.com/b/1234567/8901234.html`（两段数字都需要） |

**只填一个平台也完全可以**（某些剧本来就只在一个平台上）。

## 7. 手动触发一次采集看结果

录入之后，默认要等 15 分钟爬虫自动跑下一轮。想立即看效果：

```
root@server:~# systemctl restart rejubang-crawler
```

重启会触发"首次启动立即采集"逻辑。监看日志：

```
root@server:~# tail -f /opt/rejubang/logs/scheduler_$(date +%Y-%m-%d).log
```

正常输出类似：
```
=== 开始热度数据采集任务 ===
[AiringCrawler] 待采集 5 条记录
  [✓] 蜜语纪@tencent: 24096.0
  [✓] 蜜语纪@iqiyi: 17868.0
  [✓] 方圆八百米@tencent: 21421.0
  ...
[AiringCrawler] 完成: 保存 5 条，失败 0 条
```

`Ctrl + C` 退出日志。

### 7.1 或不重启，用命令行触发一次

```
root@server:~# cd /opt/rejubang/backend
root@server:/opt/rejubang/backend# ./venv/bin/python test_crawl.py --run
```

## 8. 查数据库验证

```
root@server:~# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang -e "
SELECT d.title, p.short_name, h.heat_value, h.record_time
FROM heat_realtime h
JOIN dramas d ON h.drama_id = d.id
JOIN platforms p ON h.platform_id = p.id
ORDER BY h.record_time DESC LIMIT 20;
"
```

应该看到刚采的热度数据。

## 9. 定时任务（爬虫服务启动后自动运行）

| 时间 | 任务 | 说明 |
|---|---|---|
| 每 15 分钟 | 实时热度采集 | 对 `drama_platforms` 里所有在播剧采热度 |
| 每日 00:00 | 数据清洗 | 去除异常值 |
| 每日 00:30 | 日度统计 | heat_realtime → heat_daily |
| 每日 01:00 | 剧力指数 | 综合热度/社交/播放/口碑 |
| 每日 01:30 | 异动检测 | 热度暴涨剧告警 |
| 每日 03:00 | 豆瓣评分 | 更新在播剧的豆瓣分 |
| 每日 04:00 | **冷数据归档** | **超过 90 天的原始热度删除（已聚合到 heat_daily）**<br>**超过 365 天的日聚合数据删除** |
| 每日 15:00 | 日度发布 | 把昨天的数据标记为 published |

数据空间占用（参考）：
- 30 部在播剧 × 4 平台 × 每 15 分钟 = ~100 MB/年（原始数据，90 天后自动清理）
- heat_daily（保留 1 年）：~1.5 MB
- 小程序查图表：从 heat_daily 读，快且省空间
