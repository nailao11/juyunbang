# 第七册：启动服务、验证、录入在播剧

> **本册在哪操作**：**云服务器** SSH 命令行（启动服务）+ **本地浏览器**（访问管理后台）。
>
> **目标**：把 API 和爬虫服务起起来，然后**通过管理后台录入在播剧**（核心步骤），验证整条链路通畅。

---

## 1. 两个后端服务是什么

| systemd 服务名 | 作用 | 运行什么 | 端口 |
|---|---|---|---|
| `rejubang-api` | Flask API，处理小程序请求 + 提供管理后台 | `gunicorn run:app` | 127.0.0.1:5000 |
| `rejubang-crawler` | 爬虫调度器，每 15 分钟对已录入的剧采热度 | `python scheduler/task_scheduler.py` | 不监听端口 |

两者共用 `/opt/rejubang/backend/venv/` 虚拟环境和 `/opt/rejubang/backend/.env` 配置。

## 2. 创建日志目录

📍 **位置**：云服务器 · 当前目录 `/root`

```
mkdir -p /opt/rejubang/logs
```

---

## 3. 安装 systemd 服务文件

📍 **位置**：云服务器 · 起始目录 `/root`（块内会切换到 `/opt/rejubang/deploy`，下方 `cd` 已写入代码，整段复制即可）

```
cd /opt/rejubang/deploy
cp rejubang-api.service /etc/systemd/system/
cp rejubang-crawler.service /etc/systemd/system/
systemctl daemon-reload
```

---

## 4. 启动两个服务

📍 **位置**：云服务器 · 当前目录 `/root`

```
systemctl enable --now rejubang-api
systemctl enable --now rejubang-crawler
```

### 4.1 状态检查

📍 **位置**：云服务器 · 当前目录 `/root`

```
systemctl is-active rejubang-api rejubang-crawler
active
active
```

两行 `active` 就对了。若有 `failed` 看日志：

📍 **位置**：云服务器 · 当前目录 `/root`

```
journalctl -u rejubang-api -n 100 --no-pager
journalctl -u rejubang-crawler -n 100 --no-pager
```

## 6. 【关键新步骤】通过管理后台录入在播剧

> ⚠️ **本项目架构变更说明（2026-04）**：不再自动"发现"新剧，**由你手动录入在播剧清单**。工作量很小——每月新剧也就 5~10 部，录一部只要 1 分钟。
>
> **如果跳过这一步，爬虫会正常运行但没有任何剧可采，数据库永远是空的。**

### 6.1 打开管理后台

**本地电脑**浏览器访问：

```
https://<你的API域名>/admin
```

（例如 `https://api.nailao.asia/admin`）

会看到一个登录页，要求输入 Admin Token。

### 6.2 用 `ADMIN_TOKEN` 登录

- 粘贴第五册 `.env` 里的 `ADMIN_TOKEN`（默认预设值是 `zCfQR0OkgWbZwaG4iw5Vb1E3ksvGAxTpR8nGk7gd--c`）
- 点"登录"
- 成功后进入管理面板

### 6.3 录入一部在播剧

> ⚠️ 从 v5 起，**四个平台都只接受完整页面链接**（不再支持裸 ID、不再支持旧 cover URL）。
> 部署后没有任何"测试剧"会被自动导入，必须通过这里录入后才会有采集数据。

在"➕ 新增 / 更新剧集"表单里填：

| 字段 | 示例值 | 说明 |
|---|---|---|
| 剧名 | `方圆八百米` | 必填 |
| 开播日期 | `2026-04-17` | 留空默认今天 |
| 类型 | 电视剧 | 下拉选 |
| 腾讯视频 | `https://m.v.qq.com/x/m/play?cid=mzc002007tp60ap&vid=w41025my54z` | **移动播放页**，URL 必须同时包含 `cid` 和 `vid` |
| 爱奇艺 | `https://www.iqiyi.com/a_1euk1nkfz9l.html` | **PC 专辑页**，必须是 `a_` 前缀 |
| 优酷 | `https://v.youku.com/v_show/id_xxxxxxxx.html` | 完整播放页 URL |
| 芒果TV | `https://www.mgtv.com/b/{partId}/{clipId}.html` | 完整剧集页 URL（含两段数字 ID） |

四个平台**至少填一个**即可（某些剧本来就只在一个平台播）。

点击 **"先测试热度提取"** → 浏览器会显示每个平台的实时抓取结果（约 10~30 秒），并展示 debug 详情（`source_type` / `match_pattern` / `matched_snippet` / `final_url`）：

```
✓ tencent      value = 24096    rendered_text     站内热度 24096
✓ iqiyi        value = 7903     base_info_heat    7903
✓ mgtv         value = 2290000000  rendered_text  22.9亿次播放
```

看到 value > 0 即链路通畅，再点 **"保存录入"**。如果某个平台返回 0，debug 区域会显示具体原因（如 `errors: heat not found in base_info`），可据此调整链接或排查反爬。

### 6.4 哪里复制各平台的 URL？

| 平台 | 怎么拿 |
|---|---|
| 腾讯视频 | **手机浏览器**打开剧集任意一集，地址栏即 `m.v.qq.com/x/m/play?cid=...&vid=...` |
| 爱奇艺 | PC 浏览器打开剧集详情页（不是单集页），地址栏形如 `www.iqiyi.com/a_xxxxxxxxx.html` |
| 优酷 | PC 浏览器打开剧集播放页，地址栏形如 `v.youku.com/v_show/id_xxxxxxxxx.html` |
| 芒果TV | PC/移动浏览器剧集页，地址栏形如 `www.mgtv.com/b/{partId}/{clipId}.html`（两段数字都需要） |

## 7. 手动触发一次采集看结果

录入之后，默认要等 15 分钟爬虫自动跑下一轮。想立即看效果：

📍 **位置**：云服务器 · 当前目录 `/root`

```
systemctl restart rejubang-crawler
```

重启会触发"首次启动立即采集"逻辑。监看日志：

📍 **位置**：云服务器 · 当前目录 `/root`

```
tail -f /opt/rejubang/logs/scheduler_$(date +%Y-%m-%d).log
```

正常输出类似：
```
=== 开始热度数据采集任务 ===
[AiringCrawler] 待采集 N 条记录
  [✓] 你录的剧名@tencent: 24096.0
  [✓] 你录的剧名@iqiyi:   7903.0
  ...
[AiringCrawler] 完成: 保存 N 条，失败 0 条
```

`Ctrl + C` 退出日志。

### 7.1 或不重启，用命令行触发一次

📍 **位置**：云服务器 · 起始目录 `/root`（块内会切换到 `/opt/rejubang/backend`，下方 `cd` 已写入代码，整段复制即可）

cd /opt/rejubang/backend
./venv/bin/python test_crawl.py --run

## 8. 查数据库验证

📍 **位置**：云服务器 · 当前目录 `/root`

```
mariadb -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang -e "
SELECT d.title, p.short_name, h.heat_value, h.record_time
FROM heat_realtime h
JOIN dramas d ON h.drama_id = d.id
JOIN platforms p ON h.platform_id = p.id
ORDER BY h.record_time DESC LIMIT 20;
"
```

应该看到刚采的热度数据。

## 9. 定时任务（爬虫服务启动后自动运行）

| 时间 | 任务 | 说明 |
|---|---|---|
| 每 15 分钟 | 实时热度采集 | 对 `drama_platforms` 里所有在播剧采热度 |
| 每日 00:00 | 数据清洗 | 去除异常值 |
| 每日 00:30 | 日度统计 | heat_realtime → heat_daily |
| 每日 01:00 | 剧力指数 | 综合热度/社交/播放/口碑 |
| 每日 01:30 | 异动检测 | 热度暴涨剧告警 |
| 每日 03:00 | 豆瓣评分 | 更新在播剧的豆瓣分 |
| 每日 04:00 | **冷数据归档** | **超过 90 天的原始热度删除（已聚合到 heat_daily）**<br>**超过 365 天的日聚合数据删除** |
| 每日 15:00 | 日度发布 | 把昨天的数据标记为 published |

数据空间占用（参考）：
- 30 部在播剧 × 4 平台 × 每 15 分钟 = ~100 MB/年（原始数据，90 天后自动清理）
- heat_daily（保留 1 年）：~1.5 MB
- 小程序查图表：从 heat_daily 读，快且省空间
