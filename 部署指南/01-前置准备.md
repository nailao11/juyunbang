# 第一册：前置准备

> **本册在哪操作**：**本地电脑** + **各服务商的网页后台**（不涉及服务器命令行）。
>
> **目标**：把部署前所有"需要买的、需要申请的、需要填表的"事情全部搞定。完成本册后你手里应该有：
> - 一台可 SSH 登录的云服务器（拿到公网 IP + root 密码）
> - 一个已解析到服务器的域名（主域名 + api 子域名）
> - 一个微信小程序账号（AppID + AppSecret）
> - （可选）一个七牛云账号（AccessKey / SecretKey / Bucket / CDN 域名）

---

## 1. 购买云服务器

### 1.1 推荐配置

| 项目 | 推荐规格 | 说明 |
|---|---|---|
| CPU | 4 核 | Playwright 爬虫对 CPU 敏感 |
| 内存 | 4 GB | 会再加 2GB Swap 托底 |
| 硬盘 | 40 GB SSD | 预留日志和备份空间 |
| 操作系统 | **Debian 13 (Trixie)** 或 Debian 12 (Bookworm) | 两版本都支持，推荐 13 |
| 带宽 | 3~5 Mbps | 展示类小程序够用 |

> ⚠️ **系统一定要选 Debian 13 或 12**。如果选 Ubuntu/CentOS 某些包管理命令会不一样，本文档不保证兼容。
>
> **Debian 13 与 12 的部署差异**：只有 **Playwright 手动装依赖时的包名不同**（t64 后缀）。`requirements.txt` 里的 Python 依赖版本已经同时兼容两个系统。

### 1.2 选哪家？

- **国内用户 + 小程序商用**：阿里云 / 腾讯云 / 华为云（必须备案，后面讲）
- **个人学习 / 快速试跑**：DigitalOcean / Vultr / Linode（境外，无需备案）

### 1.3 购买后你应该拿到的信息

在服务商控制台的"实例详情"页可以看到：

- **公网 IP**：类似 `123.45.67.89`，**本册和后续所有文档都会用它**
- **root 登录密码**（或 SSH 私钥文件）：购买时自己设的
- **22 端口是否放行**：默认放行，若没放行需要在"安全组/防火墙"页放行 22/80/443 三个端口

> ✅ **动作点**：把公网 IP 和 root 密码记在便签或密码管理器里，后面每一步都要用。

### 1.4 开放必要的端口

在服务商控制台的 **安全组 / 防火墙** 页，确认以下入站规则已放行：

| 端口 | 协议 | 用途 |
|---|---|---|
| 22 | TCP | SSH 登录服务器 |
| 80 | TCP | HTTP（Certbot 申请证书要用） |
| 443 | TCP | HTTPS（小程序访问 API） |

其他端口（3306、6379、5000 等）**一律不要对外放行**，仅内网使用。

## 2. 购买域名

### 2.1 可以用什么后缀

`.cn` / `.com` / `.net` / `.asia` / `.xyz` 都可以。**微信小程序对后缀没有限制**，只要能解析并配 HTTPS 即可。

### 2.2 在哪买

- 国内：阿里云万网、腾讯云、华为云
- 境外：Cloudflare、Namesilo、GoDaddy

### 2.3 ICP 备案（只有国内服务器需要）

| 场景 | 是否需要备案 |
|---|---|
| 服务器是**国内**节点（阿里云国内、腾讯云国内等） | **必须备案**，周期 10~20 个工作日 |
| 服务器是**境外**节点（DigitalOcean、Vultr、AWS 海外） | **不需要备案**，DNS 生效即可 |

> 📌 **备案期间你可以先在本地用微信开发者工具开发调试，等备案过了再上服务器部署。**
> 备案流程在各服务商控制台的"备案"栏目有详细引导，本文不展开。

---

## 3. 配置域名 DNS 解析

假设你买的域名是 `example.com`（后续文档示例会用 `nailao.asia`，替换成你自己的即可）。

### 3.1 在域名服务商控制台添加两条 A 记录

登录你买域名的服务商控制台（阿里云/腾讯云/Cloudflare 等）→ 找到"域名解析" / "DNS 管理" → 添加记录：

| 主机记录 | 记录类型 | 记录值 | TTL |
|---|---|---|---|
| `@` | A | `<你的服务器IP>` | 600（或默认） |
| `api` | A | `<你的服务器IP>` | 600（或默认） |

- **`@` 表示主域名本身**（即 `example.com`）
- **`api` 表示子域名**（即 `api.example.com`）

### 3.2 等待 DNS 生效（5~30 分钟）

在 **本地电脑** 的命令行里验证（Windows 用 PowerShell，Mac/Linux 用终端）：

📍 **位置**：本地电脑

```
ping example.com
ping api.example.com
```

只要两条命令都解析到**你服务器的公网 IP**，就成功了。能不能 ping 通无所谓（有些服务商默认禁 ping），看"解析到的 IP"就行。

## 4. 注册微信小程序

### 4.1 注册账号

浏览器打开 https://mp.weixin.qq.com → 点击"立即注册" → 选择 **"小程序"** → 按提示完成邮箱验证、主体信息填写（个人主体也可以）。

### 4.2 获取 AppID 和 AppSecret

登录后 → **开发管理** → **开发设置** → 页面顶部能看到：

- **AppID (小程序ID)**：形如 `wx` 开头的字符串，公开可见
- **AppSecret (小程序密钥)**：点击"生成"或"重置"按钮查看，**只显示一次，务必保存**

> ✅ **动作点**：把 AppID 和 AppSecret 记下来，后面 [05-环境变量配置.md](./05-环境变量配置.md) 要用。

### 4.3 暂不配置服务器合法域名

**现在不要着急填合法域名**，因为此时你的 HTTPS 证书还没申请。后面 [08-小程序配置与发布.md](./08-小程序配置与发布.md) 会专门讲什么时候配、配什么。

## 5. （可选）注册七牛云

### 5.1 七牛云是干什么的？

七牛云用来存储 **用户上传的图片**（头像、意见反馈的截图）。小程序不能把图片直接传到你自己的服务器（容量 + 带宽成本高），需要专门的对象存储服务。

> ⚠️ **如果暂时不需要头像和图片上传功能，本节可以整个跳过。** 等后面想用再回来配即可，不影响热度采集和榜单展示。

### 5.2 注册与申请

1. 浏览器打开 https://portal.qiniu.com → 注册并完成实名认证
2. 控制台左侧 **"对象存储 Kodo"** → **"空间管理"** → **"新建存储空间"**
   - 名称：`rejubang`（或自取，要记住）
   - 存储区域：选离你服务器近的
   - 访问控制：**公开空间**（图片需要直接给小程序读取）
3. 在空间详情页 → **"域名管理"** → **"绑定域名"**（需要一个已备案的子域名做 CDN，如 `cdn.example.com`）
4. 控制台右上角头像 → **"密钥管理"** → 查看 **AccessKey** 和 **SecretKey**

### 5.3 需要记下的四个值

| 名称 | 示例 | 填到 `.env` 的哪个字段 |
|---|---|---|
| AccessKey | `a1b2c3...` | `QINIU_ACCESS_KEY` |
| SecretKey | `x9y8z7...` | `QINIU_SECRET_KEY` |
| Bucket 名称 | `rejubang` | `QINIU_BUCKET` |
| CDN 域名 | `cdn.example.com` | `QINIU_DOMAIN` |

## 6. 安装 SSH 客户端（本地电脑）

SSH 客户端用来连接到云服务器的命令行。

### 6.1 根据操作系统选择

| 系统 | 推荐工具 | 备注 |
|---|---|---|
| **Windows** | [MobaXterm](https://mobaxterm.mobatek.net/) 或 [WindTerm](https://github.com/kingToolbox/WindTerm) | MobaXterm 带 SFTP 文件传输，新手首选 |
| **Mac** | 系统自带的 **终端.app** | 直接打开用 `ssh` 命令即可 |
| **Linux** | 自带终端 | 同上 |
| **安卓手机临时救急** | Termius / JuiceSSH | ⚠️ 断开重连后会回到 `/root`，要重新 `cd` |

### 6.2 首次 SSH 登录

打开 **本地电脑** 的终端/SSH 客户端，执行：

📍 **位置**：本地电脑

```
ssh root@<你的服务器IP>
```

举例（假设 IP 是 `123.45.67.89`）：

📍 **位置**：本地电脑

```
ssh root@123.45.67.89
```

第一次连接会提示：

```
The authenticity of host '123.45.67.89' can't be established.
...
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

输入 `yes` 回车，然后输入你的 root 密码（**输密码时屏幕不会显示任何字符，这是正常的**，输完回车即可）。

登录成功后会看到类似：

```
root@debian:~#
```

这就表示你已经进入服务器。后续命令默认都在这个命令行里执行。

> 💡 **小技巧**：输完长命令前先 `pwd`（显示当前目录）和 `whoami`（显示当前用户）可以避免在错的地方误操作。
