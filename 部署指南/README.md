## 部署前你需要知道的事

本项目包含两部分：
- **后端**：部署在 **Linux 云服务器** 上（Python + Flask + MariaDB + Redis + Playwright）。
- **前端**：微信小程序，部署在 **你的电脑** 上的"微信开发者工具"里编译并提交审核。

两部分通过 HTTPS 接口通信，所以你需要：
1. **一台云服务器**（4核4G，Debian 12）
2. **一个域名**（做 HTTPS + 微信小程序合法域名要求）
3. **一台本地电脑**（Windows/Mac 均可，用来跑微信开发者工具）

## 🆕 2026-04 架构变更（必读）

本项目从 v4 升级到 **v5 半自动架构**：

- ❌ **旧方案**：爬虫自动从各平台 API 发现新剧 → 匹配数据库 → 采集热度（多环节失败率高）
- ✅ **新方案**：你通过 **Web 管理后台** 录入在播剧 → 爬虫只负责按清单采热度（稳定）

**这意味着部署过程中多了一步"录入在播剧"**（不录就没数据）。详见 [07-启动服务与验证.md](./07-启动服务与验证.md) 第 6 节。好消息是每部剧只需 1 分钟，新剧一个月也就 5~10 部。

## 🆕 2026-04 操作系统支持

本指南同时支持：

| 系统 | 默认 Python | 备注 |
|---|---|---|
| **Debian 13 (Trixie)** 13.2+ | 3.13 | **推荐**。依赖版本已适配（playwright 1.55 / lxml 5.3 / cryptography 44） |
| Debian 12 (Bookworm) | 3.11 | 仍然完全支持 |

**如果你服务器是纯净的 Debian 13**，直接按本指南操作即可。部署指南内的命令对两个版本都适用（个别差异会特别标注）。

> ⚠️ 如果你还在用 Debian 11 或更老版本，请先升级到 12 或 13，本项目不再测试更老的版本。

## 一些必须先理解的约定

阅读文档时请记住以下几个约定，之后不再重复说明：

### 1. 命令工作目录约定（重要）

本指南**每个代码块上方**都用一行 `📍 位置` 明确标注命令在哪台机器、哪个目录下执行：

```
📍 **位置**：云服务器 · 当前目录 `/opt/rejubang/backend`
```

或

```
📍 **位置**：本地电脑
```

含义：

| 标注 | 含义 |
|---|---|
| `本地电脑` | 在你自己电脑的终端里执行（Windows PowerShell / Mac 终端） |
| `云服务器 · 当前目录 /xxx` | 在云服务器 SSH 里执行，**整段命令都在 `/xxx` 这个目录下** |
| `云服务器 · 起始目录 /xxx（块内会切换到 /yyy）` | 块内会切目录，**已经把 `cd` 写进代码里**，整段复制即可，不会跑错位置 |

**复制规则**：
- ✅ 直接复制整个代码块的内容粘贴到终端，包括代码块里出现的 `cd ...` 行（脚本已经把目录切换写进去了）
- ❌ 不要只复制看起来"业务相关"的那一行，可能会缺少前面的 `cd`，导致在错误目录执行
- 命令前不再有 `root@server:/path#` 之类的提示符 —— 工作目录信息全部写在代码块**上方**的 `📍 位置` 行里，复制时不会粘到无效字符

### 2. 需要你替换的占位符

文档里所有尖括号包裹的内容 `<xxx>` 都是占位符，需要替换成你自己的值：

| 占位符 | 含义 | 举例 |
|---|---|---|
| `<你的服务器IP>` | 云服务器公网 IP | `123.45.67.89` |
| `<你的域名>` | 你买的主域名 | `nailao.asia` |
| `<你的API域名>` | 通常是 `api.<你的域名>` | `api.nailao.asia` |
| `<你的小程序AppID>` | 微信小程序 AppID | `wx1234567890abcdef` |

**替换时不要保留尖括号。** 比如 `<你的域名>` 替换后应该是 `nailao.asia`，不是 `<nailao.asia>`。

### 3. 已经预生成好的密钥

以下密钥与密码文档里已经帮你**生成好了固定值**，直接复制粘贴即可，不需要你再重新生成、也不需要你记住：

- `SECRET_KEY`（Flask 会话密钥）
- `JWT_SECRET_KEY`（JWT 签名密钥）
- `DB_PASSWORD`（项目数据库用户密码）

这些值只在服务器本地使用，不对外暴露，用预设值不影响安全性。

### 4. 随时可以中断重来

部署流程里任何一步出错都可以定位到 [10-常见问题FAQ.md](./10-常见问题FAQ.md) 或重新执行对应步骤。已完成的步骤（如装系统包、建库）重复执行大多数是幂等的（不会造成破坏）。
