# 第五册：环境变量配置（`.env`）

> **本册在哪操作**：**云服务器** SSH 命令行，全程在 `/opt/rejubang/backend/` 目录下。
>
> **目标**：把 `.env` 配置文件填好。后端程序启动时会读取这个文件的内容。

---

## 0. 本册特别说明

原来的部署指南在这一步最容易让新手迷惑——要自己跑命令生成随机密钥、要记几个不同的值、要在 vi 里编辑……很容易漏填或填错。

**本文档的做法是：**
- **所有不需要你记忆的"随机密钥"（SECRET_KEY、JWT_SECRET_KEY）已经预先生成好了**，直接复制粘贴即可。
- **数据库密码**用第四册已经给你用的那个 `i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA`。
- **只有微信 AppID/Secret 和七牛云 4 个值需要你自己填**（第一册已准备好）。

```
root@server:~# cd /opt/rejubang/backend
root@server:/opt/rejubang/backend# pwd
/opt/rejubang/backend
```

## 2. 从模板复制一份 `.env`

```
root@server:/opt/rejubang/backend# cp .env.example .env
root@server:/opt/rejubang/backend# ls -la .env
-rw-r--r-- 1 root root ... .env
```

`.env.example` 是模板（不会被提交到 git，但项目自带），`.env` 是你实际用的配置（已在 `.gitignore`，不会误传）。

## 3. 一次性写入 `.env` 的完整内容

### 3.1 方案 A（推荐）：用 `cat > .env` 一次性写入

**这种方式最不容易出错**。直接复制以下完整命令块，粘贴到终端，按回车：

```bash
cat > /opt/rejubang/backend/.env <<'ENVEOF'
# ========== Flask 基础配置 ==========
FLASK_ENV=production
SECRET_KEY=324b6e877ce14f09b61aa25adb9aeae9d7f1914c21153ea9634e8f5421bf1864

# ========== MariaDB 数据库 ==========
DB_HOST=localhost
DB_PORT=3306
DB_NAME=rejubang
DB_USER=rejubang
DB_PASSWORD=i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA

# ========== Redis 缓存 ==========
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0

# ========== 微信小程序（必须填自己的）==========
WX_APPID=替换为你的小程序AppID
WX_SECRET=替换为你的小程序AppSecret

# ========== 七牛云对象存储（可选，不用填"需要填"字样即可跳过）==========
QINIU_ACCESS_KEY=替换为你的七牛云AccessKey
QINIU_SECRET_KEY=替换为你的七牛云SecretKey
QINIU_BUCKET=替换为你的七牛云Bucket名称
QINIU_DOMAIN=替换为你的七牛云CDN域名

# ========== JWT 认证 ==========
JWT_SECRET_KEY=2926c6e52e3176d567e999acba0132c4acbb014e1dab71e6a0785697f2ef5e48
JWT_ACCESS_TOKEN_EXPIRES=86400
ENVEOF
```

> 💡 **`<<'ENVEOF' ... ENVEOF` 是 heredoc 语法**，让 shell 把两行 `ENVEOF` 之间的所有内容当成文件内容写入 `.env`。单引号 `'ENVEOF'` 很重要，**千万别去掉**，否则里面的 `$` 字符会被 shell 解析。

执行后 `.env` 就存在了，接下来只需编辑替换"AppID / AppSecret"和"七牛云四项"。

### 3.2 方案 B：手动用 vi/nano 编辑

如果你更习惯手动编辑：

```
root@server:/opt/rejubang/backend# nano .env
```

然后把下面这段完整粘贴进去：

```ini
# ========== Flask 基础配置 ==========
FLASK_ENV=production
SECRET_KEY=324b6e877ce14f09b61aa25adb9aeae9d7f1914c21153ea9634e8f5421bf1864

# ========== MariaDB 数据库 ==========
DB_HOST=localhost
DB_PORT=3306
DB_NAME=rejubang
DB_USER=rejubang
DB_PASSWORD=i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA

# ========== Redis 缓存 ==========
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0

# ========== 微信小程序（必须填自己的）==========
WX_APPID=替换为你的小程序AppID
WX_SECRET=替换为你的小程序AppSecret

# ========== 七牛云对象存储（可选）==========
QINIU_ACCESS_KEY=替换为你的七牛云AccessKey
QINIU_SECRET_KEY=替换为你的七牛云SecretKey
QINIU_BUCKET=替换为你的七牛云Bucket名称
QINIU_DOMAIN=替换为你的七牛云CDN域名

# ========== JWT 认证 ==========
JWT_SECRET_KEY=2926c6e52e3176d567e999acba0132c4acbb014e1dab71e6a0785697f2ef5e48
JWT_ACCESS_TOKEN_EXPIRES=86400
```

**nano 的保存操作**：`Ctrl + O` → 回车 → `Ctrl + X` 退出。

**vi 的保存操作**：`ESC` → 输入 `:wq` → 回车。

## 4. 替换微信和七牛云的实际值

### 4.1 用 `sed` 一次性替换（推荐，不会手抖）

把下面的命令里的**尖括号占位符**替换成你在第一册记下的真实值，然后一次性执行：

```bash
# 微信小程序（必须替换）
sed -i 's|替换为你的小程序AppID|<你的小程序AppID>|' /opt/rejubang/backend/.env
sed -i 's|替换为你的小程序AppSecret|<你的小程序AppSecret>|' /opt/rejubang/backend/.env

# 七牛云（如果不用七牛云，跳过这 4 行）
sed -i 's|替换为你的七牛云AccessKey|<你的七牛云AccessKey>|' /opt/rejubang/backend/.env
sed -i 's|替换为你的七牛云SecretKey|<你的七牛云SecretKey>|' /opt/rejubang/backend/.env
sed -i 's|替换为你的七牛云Bucket名称|<你的七牛云Bucket名称>|' /opt/rejubang/backend/.env
sed -i 's|替换为你的七牛云CDN域名|<你的七牛云CDN域名>|' /opt/rejubang/backend/.env
```

**具体例子**（假设你的 AppID 是 `wx1234567890abcdef`）：

```bash
sed -i 's|替换为你的小程序AppID|wx1234567890abcdef|' /opt/rejubang/backend/.env
```

> ⚠️ **为什么用 `|` 做分隔符而不是 `/`？** 因为七牛云域名里可能有 `/`，用 `|` 避免冲突。

### 4.2 或者用 `nano .env` 手工改

```
root@server:/opt/rejubang/backend# nano .env
```

按 `Ctrl + W` 可以搜索 "替换" 二字快速定位需要改的行。改完 `Ctrl + O` → 回车 → `Ctrl + X`。

## 5. 验证 `.env` 内容

### 5.1 查看完整内容

```
root@server:/opt/rejubang/backend# cat .env
```

逐行检查：**不能再有"替换为..."字样**（除非你跳过了七牛云）。

### 5.2 验证关键字段

```
root@server:/opt/rejubang/backend# grep -E "^(DB_PASSWORD|WX_APPID|SECRET_KEY|JWT_SECRET_KEY)=" .env
```

应该看到类似：

```
SECRET_KEY=324b6e877ce14f09b61aa25adb9aeae9d7f1914c21153ea9634e8f5421bf1864
DB_PASSWORD=i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA
WX_APPID=wx1234567890abcdef
JWT_SECRET_KEY=2926c6e52e3176d567e999acba0132c4acbb014e1dab71e6a0785697f2ef5e48
```

### 5.3 限制 `.env` 权限（安全加固）

`.env` 包含数据库密码和小程序 secret，设为只有 root 能读：

```
root@server:/opt/rejubang/backend# chmod 600 .env
root@server:/opt/rejubang/backend# ls -la .env
-rw------- 1 root root ... .env
```

看到权限是 `-rw-------` 就对了。

## 6. 一次性连通测试（确认 `.env` 能被正确读取）

在 `/opt/rejubang/backend/` 目录下执行：

```
root@server:/opt/rejubang/backend# ./venv/bin/python -c "
import os
from dotenv import load_dotenv
load_dotenv()
import pymysql, redis

# 测试数据库
conn = pymysql.connect(host=os.getenv('DB_HOST'), port=int(os.getenv('DB_PORT')),
    user=os.getenv('DB_USER'), password=os.getenv('DB_PASSWORD'),
    database=os.getenv('DB_NAME'), charset='utf8mb4')
with conn.cursor() as cur:
    cur.execute('SELECT COUNT(*) FROM platforms')
    print('MariaDB OK, platforms =', cur.fetchone()[0])
conn.close()

# 测试 Redis
r = redis.Redis(host=os.getenv('REDIS_HOST'), port=int(os.getenv('REDIS_PORT')),
    db=int(os.getenv('REDIS_DB')), decode_responses=True)
print('Redis OK, ping =', r.ping())
"
```

看到：

```
MariaDB OK, platforms = 4
Redis OK, ping = True
```

就表示 `.env` 里的数据库密码、Redis 配置全部正确。

**如果报 `Access denied for user 'rejubang'...`**：`.env` 里 `DB_PASSWORD` 和第四册创建的密码不一致，重新检查。

**如果报 `Can't connect to MySQL server`**：MariaDB 没启动，`systemctl start mariadb`。
