# 第二册：系统环境安装

> **本册在哪操作**：**云服务器** SSH 命令行。
>
> **目标**：在干净的 Debian 13 (Trixie) 或 Debian 12 (Bookworm) 系统上装好所有基础依赖（Python、Nginx、MariaDB、Redis、Certbot），并配置 2GB Swap 防止内存溢出。
>
> **两个系统命令完全一致**，只有最终版本号不同：
> - Debian 13：Python 3.13、MariaDB 11.8
> - Debian 12：Python 3.11、MariaDB 10.11

---

## 0. 执行前检查

本册所有命令都在**服务器**上执行，用 root 账户。先确认：

📍 **位置**：云服务器 · 当前目录 `/root`

```
whoami
root

pwd
/root
```

如果 `whoami` 不是 `root`，执行 `sudo -i` 切换到 root。

---

## 1. 更新系统包索引

apt 是 Debian 的包管理器，先让它拉取最新的包列表：

📍 **位置**：云服务器 · 当前目录 `/root`

```
apt update && apt upgrade -y
```

- `apt update`：刷新"有哪些包可以装"的清单
- `apt upgrade -y`：把已装的包升级到最新版（`-y` 表示自动回答"是"，遇到确认就自动同意）

这一步可能会跑 1~3 分钟，结束后会回到命令提示符。

### 如果升级过程中弹出蓝色交互框

遇到类似 "Configuring libc6... Restart services? [Yes/No]" 的蓝色弹窗：

- 方向键选 **Yes** 或直接回车（默认就是 Yes）

---

## 2. 一次性安装所有基础软件

下面这条命令会把后续需要的所有服务端软件一次性装全：

📍 **位置**：云服务器 · 当前目录 `/root`

```
apt install -y \
  python3 python3-venv python3-pip \
  git curl wget \
  nginx \
  mariadb-server mariadb-client \
  redis-server \
  certbot python3-certbot-nginx
```

每个包的作用：

| 软件 | 作用 |
|---|---|
| `python3` / `python3-venv` / `python3-pip` | Python 运行时 + 虚拟环境工具 + 包管理器 |
| `git` / `curl` / `wget` | 拉代码、下载文件的基础工具 |
| `nginx` | 反向代理（把 HTTPS 请求转发到 Flask） |
| `mariadb-server` / `mariadb-client` | 数据库（MariaDB 是 MySQL 的兼容分支，Debian 默认打包） |
| `redis-server` | Redis 缓存 |
| `certbot` / `python3-certbot-nginx` | Let's Encrypt 免费 HTTPS 证书工具 |

安装过程约 2~5 分钟。结束后回到命令提示符。

### 如果报 `E: Unable to locate package ...`

说明 `apt update` 那步没成功。重新跑一次：

📍 **位置**：云服务器 · 当前目录 `/root`

```
apt update
apt install -y python3 python3-venv python3-pip git curl wget nginx mariadb-server mariadb-client redis-server certbot python3-certbot-nginx
```

## 3. 验证全部装好

逐条执行，每条应该都能返回版本号（不能报 `command not found`）：

📍 **位置**：云服务器 · 当前目录 `/root`

```
python3 --version
Python 3.13.x   ← Debian 13 默认；Debian 12 是 3.11.x（≥ 3.9 就行）

pip3 --version
pip 24.x / 23.x

git --version
git version 2.45.x（Debian 13）/ 2.39.x（Debian 12）

nginx -v
nginx version: nginx/1.26.x（Debian 13）/ 1.22.x（Debian 12）

mysql --version
# Debian 13: mariadb  Ver 11.8.x
# Debian 12: mysql  Ver 15.1 Distrib 10.11.x-MariaDB

redis-cli --version
redis-cli 7.x

certbot --version
certbot 2.x.x
```

只要每条都有版本号，这一步就完成。任何一条报 `command not found`，回到第 2 步重装对应的包。

## 4. 配置 2GB Swap

### 为什么要配 Swap？

- Playwright 启动 Chromium 浏览器，单实例占 300~500 MB 内存
- 服务器同时跑 Flask + MariaDB + Redis + Chromium，峰值可能超过 4GB
- 物理内存吃满后，Linux 会随机"杀进程"（OOM Killer），最先死的通常是占内存最多的 MariaDB
- 加 2GB Swap（把磁盘当临时内存用）就能撑过峰值

### 4.1 检查是否已有 Swap

📍 **位置**：云服务器 · 当前目录 `/root`

```
swapon --show
```

**如果有输出**（例如 `NAME TYPE SIZE ...`）：说明已经有 Swap，**跳到第 5 步**。

**如果没有任何输出**：继续下面的 4.2。

### 4.2 创建 2GB Swap 文件

逐条执行（**当前目录无所谓**，这些命令用的都是绝对路径）：

📍 **位置**：云服务器 · 当前目录 `/root`

```
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
```

每条的作用：

| 命令 | 作用 |
|---|---|
| `fallocate -l 2G /swapfile` | 在 `/` 根目录下创建 2GB 的空文件 |
| `chmod 600 /swapfile` | 限制只有 root 能读写（Swap 文件必须这个权限） |
| `mkswap /swapfile` | 把这个文件格式化成 Swap 格式 |
| `swapon /swapfile` | 启用这个 Swap |

### 4.3 让 Swap 开机自动挂载

追加一行到 `/etc/fstab`，这样服务器重启后 Swap 会自动启用：

📍 **位置**：云服务器 · 当前目录 `/root`

```
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

### 4.4 验证 Swap 生效

📍 **位置**：云服务器 · 当前目录 `/root`

```
free -h
               total        used        free      shared  buff/cache   available
Mem:           3.8Gi       400Mi       2.8Gi        10Mi       700Mi       3.3Gi
Swap:          2.0Gi          0B       2.0Gi
```

**Swap 行显示 2.0Gi** 就成功了。

## 5. 启动并设置开机自启

MariaDB 和 Redis 装完后需要启动，Nginx 装完通常已经启动。执行以下命令全部设置开机自启：

📍 **位置**：云服务器 · 当前目录 `/root`

```
systemctl enable --now mariadb
systemctl enable --now redis-server
systemctl enable --now nginx
```

`enable --now` 等于 `enable` + `start` 两个动作。

验证三个服务都在运行：

📍 **位置**：云服务器 · 当前目录 `/root`

```
systemctl is-active mariadb
active

systemctl is-active redis-server
active

systemctl is-active nginx
active
```

三条都返回 `active` 即可。

> 💡 **MariaDB 初始化设置（设 root 密码等）** 和 **Redis 测试** 放到 [04-数据库配置.md](./04-数据库配置.md) 里统一做，这里先让它们跑起来即可。
