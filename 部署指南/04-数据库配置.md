## 📌 关于命令名（重要）

**Debian 13 (Trixie) 起的 MariaDB 11.x 已经停止提供 `mysql_*` 系列兼容命令**。本册以及后续部署指南统一使用规范的 `mariadb-*` 命名：

| 老命令（Debian 12 仍可用） | 新命令（Debian 12/13 都可用） | 用途 |
|---|---|---|
| `mysql_secure_installation` | **`mariadb-secure-installation`** | 初始化向导（设置 root 密码等） |
| `mysql -u …` | **`mariadb -u …`** | 命令行客户端 |
| `mysql --version` | **`mariadb --version`** | 查版本 |
| `mysqldump` | **`mariadb-dump`** | 备份 |
| `mysqld_safe` | **`mariadbd-safe`** | 救援模式启动数据库 |
| `pkill mysqld` | **`pkill mariadbd`** | 杀掉数据库进程 |

> 如果你执行 `mariadb-*` 命令报 `command not found`，说明 `mariadb-client` 包没装好，回 [02-系统环境安装.md 第 2 步](./02-系统环境安装.md#2-一次性安装所有基础软件) 重装。

## 2. 初始化 MariaDB（设置 root 密码）

Debian 12/13 的 MariaDB 默认允许 root 账户在本机通过 Unix socket **免密**登录，但这不够安全。我们用官方工具做初始化：

📍 **位置**：云服务器 · 当前目录 `/root`

```
mariadb-secure-installation
```

它会**逐条问你问题**，**严格按照下面的回答**即可：

### 第 1 问：输入当前 root 密码

```
Enter current password for root (enter for none):
```

**直接回车**（首次安装没有密码）。

### 第 2 问：是否切换到 `unix_socket` 认证？

```
Switch to unix_socket authentication [Y/n]
```

回答 **`n`** 并回车（我们要用密码认证，不用 unix_socket，不然配 `.env` 会麻烦）。

### 第 3 问：是否设置 root 密码？

```
Change the root password? [Y/n]
```

回答 **`Y`** 并回车。

接着会提示：

```
New password:
Re-enter new password:
```

**输入密码**：`RootDB_Pass_2026_NaiLao!` （输入时屏幕不会显示任何字符，这是正常的）

**再次输入确认**：`RootDB_Pass_2026_NaiLao!`

### 第 4 问：删除匿名用户？

```
Remove anonymous users? [Y/n]
```

回答 **`Y`**。

### 第 5 问：禁止 root 远程登录？

```
Disallow root login remotely? [Y/n]
```

回答 **`Y`**。

### 第 6 问：删除测试数据库？

```
Remove test database and access to it? [Y/n]
```

回答 **`Y`**。

### 第 7 问：重新加载权限？

```
Reload privilege tables now? [Y/n]
```

回答 **`Y`**。

结束后会看到 `All done!`，表示初始化完成。

### 2.1 验证能用密码登录 root

📍 **位置**：云服务器 · 当前目录 `/root`

```
mariadb -u root -p'RootDB_Pass_2026_NaiLao!' -e "SELECT VERSION();"
+--------------------------+
| VERSION()                |
+--------------------------+
| 11.8.x-MariaDB-0+deb13u1 |   ← Debian 13 示例（版本号略有差别属正常）
+--------------------------+
```

能返回版本号就说明 root 密码设置成功（Debian 12 上版本号会是 `10.11.x-MariaDB-...`）。

> 💡 **命令里 `-p` 和密码之间不能有空格**（`-p'密码'`），加空格客户端会把空格之后的内容当成数据库名。

## 3. 创建项目数据库和用户

### 3.1 登录 MariaDB

📍 **位置**：云服务器 · 当前目录 `/root`

```
mariadb -u root -p'RootDB_Pass_2026_NaiLao!'
```

登录成功后提示符会变成：

```
MariaDB [(none)]>
```

这说明你在 MariaDB 交互界面里，不是 Linux shell 了。

### 3.2 建库、建用户、授权（在 MariaDB 交互界面里执行）

**一次性复制以下 4 行，粘贴到 MariaDB 提示符后回车**：

```sql
CREATE DATABASE rejubang DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'rejubang'@'localhost' IDENTIFIED BY 'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA';
GRANT ALL PRIVILEGES ON rejubang.* TO 'rejubang'@'localhost';
FLUSH PRIVILEGES;
```

每条后面应该显示 `Query OK, 0 rows affected`。

每条的含义：

| 语句 | 作用 |
|---|---|
| `CREATE DATABASE rejubang ... utf8mb4 ...` | 创建名为 `rejubang` 的数据库，字符集 utf8mb4（支持中文和表情符） |
| `CREATE USER 'rejubang'@'localhost' IDENTIFIED BY '...'` | 创建一个只能从**本机**登录的用户 |
| `GRANT ALL PRIVILEGES ON rejubang.* TO 'rejubang'@'localhost'` | 把 `rejubang` 数据库的所有权限授给这个用户 |
| `FLUSH PRIVILEGES` | 立即生效 |

### 3.3 退出 MariaDB 交互

```sql
EXIT;
```

终端会回到 Linux shell（`MariaDB [(none)]>` 不再出现），后续命令都在普通 shell 中执行。

### 3.4 验证项目用户可以登录

📍 **位置**：云服务器 · 当前目录 `/root`

```
mariadb -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang -e "SELECT DATABASE();"
+------------+
| DATABASE() |
+------------+
| rejubang   |
+------------+
```

能返回 `rejubang`，说明项目用户已建好。

## 4. 导入表结构和初始数据

项目的 SQL 文件在 `/opt/rejubang/backend/migrations/` 下。

### 4.1 切到 migrations 目录（不是必须，但更直观）

📍 **位置**：云服务器 · 起始目录 `/root`（块内会切换到 `/opt/rejubang/backend/migrations`，下方 `cd` 已写入代码，整段复制即可）

```
cd /opt/rejubang/backend/migrations
ls
schema.sql  seed_data.sql
```

- `schema.sql`：14 张表的建表语句
- `seed_data.sql`：初始数据，**只包含 4 个视频平台和系统配置项**，不再预置任何示例剧集

### 4.2 导入表结构

📍 **位置**：云服务器 · 当前目录 `/opt/rejubang/backend/migrations`

```
mariadb -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang < schema.sql
```

没有任何输出就是成功（Unix 风格：无消息即好消息）。

### 4.3 导入初始数据（平台 + 系统配置）

📍 **位置**：云服务器 · 当前目录 `/opt/rejubang/backend/migrations`

```
mariadb -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang < seed_data.sql
```

同样无输出即成功。

> 📌 **注意**：从 v5 起，部署不会自动导入任何"测试剧"或"样例数据"。部署完成后必须通过管理后台 `/admin` 录入完整页面链接的剧集才会有采集数据。

### 4.4 验证全部 14 张表已创建

## 5. 验证 Redis

Redis 在第二册也启动过了，这里确认能正常通信：

📍 **位置**：云服务器 · 当前目录 `/root`

```
redis-cli ping
PONG
```

返回 `PONG` 就成功了。

### 5.1 顺便看看 Redis 端口是否只监听本机

📍 **位置**：云服务器 · 当前目录 `/root`

```
ss -ltn | grep 6379
LISTEN 0  511  127.0.0.1:6379  0.0.0.0:*
```

**只监听 `127.0.0.1:6379`** 就是安全的（不对外）。如果看到 `0.0.0.0:6379`，说明 Redis 可能对公网开放了，需要编辑 `/etc/redis/redis.conf` 把 `bind 0.0.0.0` 改成 `bind 127.0.0.1` 再 `systemctl restart redis-server`。
