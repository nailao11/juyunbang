## 2. 初始化 MariaDB（设置 root 密码）

Debian 12 的 MariaDB 默认允许 root 账户本地**免密**登录（通过 Unix socket），但这不够安全。我们用官方工具做初始化：

```
root@server:~# mysql_secure_installation
```

它会**逐条问你问题**，**严格按照下面的回答**即可：

### 第 1 问：输入当前 root 密码

```
Enter current password for root (enter for none):
```

**直接回车**（首次安装没有密码）。

### 第 2 问：是否切换到 `unix_socket` 认证？

```
Switch to unix_socket authentication [Y/n]
```

回答 **`n`** 并回车（我们要用密码认证，不用 unix_socket，不然配 `.env` 会麻烦）。

### 第 3 问：是否设置 root 密码？

```
Change the root password? [Y/n]
```

回答 **`Y`** 并回车。

接着会提示：

```
New password:
Re-enter new password:
```

**输入密码**：`RootDB_Pass_2026_NaiLao!` （输入时屏幕不会显示任何字符，这是正常的）

**再次输入确认**：`RootDB_Pass_2026_NaiLao!`

### 第 4 问：删除匿名用户？

```
Remove anonymous users? [Y/n]
```

回答 **`Y`**。

### 第 5 问：禁止 root 远程登录？

```
Disallow root login remotely? [Y/n]
```

回答 **`Y`**。

### 第 6 问：删除测试数据库？

```
Remove test database and access to it? [Y/n]
```

回答 **`Y`**。

### 第 7 问：重新加载权限？

```
Reload privilege tables now? [Y/n]
```

回答 **`Y`**。

结束后会看到 `All done!`，表示初始化完成。

### 2.1 验证能用密码登录 root

```
root@server:~# mysql -u root -p'RootDB_Pass_2026_NaiLao!' -e "SELECT VERSION();"
+-------------------------+
| VERSION()               |
+-------------------------+
| 10.11.x-MariaDB-0+deb12u1 |
+-------------------------+
```

能返回版本号就说明 root 密码设置成功。

> 💡 **命令里 `-p` 和密码之间不能有空格**（`-p'密码'`），加空格 MySQL 会以为密码是数据库名。

## 3. 创建项目数据库和用户

### 3.1 登录 MariaDB

```
root@server:~# mysql -u root -p'RootDB_Pass_2026_NaiLao!'
```

登录成功后提示符会变成：

```
MariaDB [(none)]>
```

这说明你在 MariaDB 交互界面里，不是 Linux shell 了。

### 3.2 建库、建用户、授权（在 MariaDB 交互界面里执行）

**一次性复制以下 4 行，粘贴到 MariaDB 提示符后回车**：

```sql
CREATE DATABASE rejubang DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'rejubang'@'localhost' IDENTIFIED BY 'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA';
GRANT ALL PRIVILEGES ON rejubang.* TO 'rejubang'@'localhost';
FLUSH PRIVILEGES;
```

每条后面应该显示 `Query OK, 0 rows affected`。

每条的含义：

| 语句 | 作用 |
|---|---|
| `CREATE DATABASE rejubang ... utf8mb4 ...` | 创建名为 `rejubang` 的数据库，字符集 utf8mb4（支持中文和表情符） |
| `CREATE USER 'rejubang'@'localhost' IDENTIFIED BY '...'` | 创建一个只能从**本机**登录的用户 |
| `GRANT ALL PRIVILEGES ON rejubang.* TO 'rejubang'@'localhost'` | 把 `rejubang` 数据库的所有权限授给这个用户 |
| `FLUSH PRIVILEGES` | 立即生效 |

### 3.3 退出 MariaDB 交互

```sql
EXIT;
```

提示符会变回 `root@server:~#`，说明回到 Linux shell。

### 3.4 验证项目用户可以登录

```
root@server:~# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang -e "SELECT DATABASE();"
+------------+
| DATABASE() |
+------------+
| rejubang   |
+------------+
```

能返回 `rejubang`，说明项目用户已建好。

## 4. 导入表结构和初始数据

项目的 SQL 文件在 `/opt/rejubang/backend/migrations/` 下。

### 4.1 切到 migrations 目录（不是必须，但更直观）

```
root@server:~# cd /opt/rejubang/backend/migrations
root@server:/opt/rejubang/backend/migrations# ls
schema.sql  seed_data.sql
```

- `schema.sql`：14 张表的建表语句
- `seed_data.sql`：初始数据（4 个核心平台 + 若干样例剧）

### 4.2 导入表结构

```
root@server:/opt/rejubang/backend/migrations# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang < schema.sql
```

没有任何输出就是成功（Unix 风格：无消息即好消息）。

### 4.3 导入初始数据

```
root@server:/opt/rejubang/backend/migrations# mysql -u rejubang -p'i7qxW0ZUGwWBMSUqm2TdJCB3DefTwosA' rejubang < seed_data.sql
```

同样无输出即成功。

### 4.4 验证全部 14 张表已创建

## 5. 验证 Redis

Redis 在第二册也启动过了，这里确认能正常通信：

```
root@server:~# redis-cli ping
PONG
```

返回 `PONG` 就成功了。

### 5.1 顺便看看 Redis 端口是否只监听本机

```
root@server:~# ss -ltn | grep 6379
LISTEN 0  511  127.0.0.1:6379  0.0.0.0:*
```

**只监听 `127.0.0.1:6379`** 就是安全的（不对外）。如果看到 `0.0.0.0:6379`，说明 Redis 可能对公网开放了，需要编辑 `/etc/redis/redis.conf` 把 `bind 0.0.0.0` 改成 `bind 127.0.0.1` 再 `systemctl restart redis-server`。
